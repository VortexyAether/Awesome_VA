# Daily Research Radar — 2026-06-02

## Highlights

- **오늘의 신규 신호는 operator learning 자체의 표현 문제입니다.** Functional Attention은 token-wise attention을 continuous field의 functional correspondence로 다시 정의해, PDE/3D field에서 resolution/discretization 변화에 버티는 operator 구조를 노립니다.
- **Chaotic surrogate는 “불확실성까지 맞는가”가 핵심입니다.** DPC gap/KAFFEE는 finite-horizon probabilistic objective가 동역학 불변량과 local tangent uncertainty를 망가뜨릴 수 있음을 보여줘, long-rollout CFD/thermal surrogate 평가에 좋은 경고입니다.
- **Engineering automation은 surrogate의 범위가 넓어지고 있습니다.** GPU Forecasters는 LLM을 kernel generator가 아니라 GPU runtime surrogate로 쓰고, CFDTwin/NED3는 CFD/thermal data와 GUI/API tooling을 묶습니다.
- **Agentic engineering workflow는 “agent-readable infrastructure”가 경쟁력이 됩니다.** PithTrain은 throughput뿐 아니라 coding agent가 framework를 이해·수정하는 비용까지 metric으로 둡니다. CAD/CFD 자동화 툴도 같은 기준이 필요합니다.

## Added Resources

### Operator learning / dynamical-system surrogates

- [Functional Attention: From Pairwise Affinities to Functional Correspondences](https://arxiv.org/abs/2605.31559) · [project/code](https://github.com/xjffff/FUNCATTN) — Transformer operator의 token-wise attention을 adaptive basis 사이의 structured linear operator로 바꿔, PDE solving/3D segmentation/regression에서 discretization 변화에 강한 functional representation을 제안합니다. `#operator-learning` `#attention` `#PDE` `#resolution-invariant`
- [The Dynamic-Probabilistic Consistency Gap in Chaotic Surrogate Modeling](https://arxiv.org/abs/2605.31547) — chaotic surrogate에서 probabilistic objective가 dynamics와 uncertainty를 서로 일관되지 않게 만들 수 있음을 DPC gap으로 정리하고, EKF 기반 KAFFEE training으로 local Jacobian covariance propagation을 보존하려 합니다. `#chaotic-dynamics` `#surrogate` `#uncertainty` `#long-rollout`

### Engineering workflow / open-source tooling

- [GPU Forecasters: Language Models as Selective Surrogates for Kernel Runtime Optimization](https://arxiv.org/abs/2605.31464) — LLM을 GPU kernel 생성기뿐 아니라 runtime forecaster/surrogate로 써서, 실제 GPU 측정 예산 안에서 더 많은 후보를 탐색하게 합니다. SciML/HPC codegen에서 “실측은 비싸고 search budget은 커지는” 문제와 직접 연결됩니다. `#GPU` `#kernel-optimization` `#LLM` `#surrogate`
- [PithTrain: A Compact and Agent-Native MoE Training System](https://arxiv.org/abs/2605.31463) — MoE training framework를 coding agent가 이해·수정하기 쉬운 구조로 설계하고 ATE(agent-task efficiency)를 평가합니다. CFD/CAD 자동화 repo도 “사람만 쓰기 좋은 툴”이 아니라 agent-operable system으로 설계해야 한다는 좋은 메타 신호입니다. `#agentic-workflow` `#training-system` `#MoE` `#tooling`
- [CFDTwin](https://arxiv.org/abs/2605.27725) · [code](https://github.com/UARK-NED3/CFDTwin) — ANSYS Fluent 결과를 POD-NN surrogate로 만들기 위한 GUI/Python toolkit입니다. 반복 Fluent solve를 design optimization, uncertainty analysis, digital twin에 넣으려면 모델 architecture보다 workflow packaging이 먼저라는 점에서 계속 고신호입니다. `#CFD` `#Fluent` `#POD-NN` `#digital-twin`
- [Open Multimodal Datasets and Open-Source Software for Data-Driven Modeling of Multiphase Transport and Thermal Systems](https://arxiv.org/abs/2605.23037) — boiling images, acoustic/thermal measurements, high-speed video, IR thermography, CFD fields, design files와 BubbleID/SeqReg/CFDTwin/IRISApp/FlowLab류 도구를 하나의 thermal-fluid data ecosystem으로 묶습니다. `#thermal-fluid` `#dataset` `#multimodal` `#open-source`

### CAD / validated design automation

- [Physics-in-the-Loop: A Hybrid Agentic Architecture for Validated CAD Engineering Design](https://arxiv.org/abs/2605.19717) — LLM CAD generation을 knowledge-based engineering tools와 physical verification loop 안에 넣는 architecture입니다. VA의 CAD→mesh→thermal/CFD 자동화에서도 “생성”보다 artifact gate가 핵심입니다. `#CAD-agent` `#physics-validation` `#CAE` `#design-automation`
- [Self-Improving CAD Generation Agents with Finite Element Analysis as Feedback](https://arxiv.org/abs/2605.17448) — free-form engineering brief에서 multi-part STEP를 만들고 FEA로 검증/수정하는 task formulation입니다. first-attempt strict pass가 거의 안 나온다는 결과가 오히려 중요합니다: engineering agent는 repair loop 없이는 장식품입니다. `#CAD` `#FEA` `#STEP` `#validation`

## Best Find of the Day

**Functional Attention**을 오늘의 best find로 봅니다. 이유는 단순합니다. Neural operator가 CFD/thermal field를 다루려면 mesh/token 배열을 잘 외우는 수준을 넘어, field function 사이의 mapping을 안정적으로 표현해야 합니다. “attention을 functional correspondence로 본다”는 관점은 complex geometry, resolution transfer, sparse observation assimilation 같은 VA 관심사에 직접 닿습니다.

## Possible Research Ideas

1. **Functional-attention smoke test:** 2D heat equation 또는 cylinder wake에서 FNO/Transformer operator/Functional Attention을 resolution transfer와 sparse observation 조건으로 비교한다.
2. **Surrogate uncertainty gate:** DPC gap을 참고해 CFD surrogate 평가에 short-horizon error뿐 아니라 Jacobian/local sensitivity, invariant drift, calibrated uncertainty를 넣는다.
3. **Agent-operable CFD repo checklist:** PithTrain의 ATE 관점으로 CFD/thermal automation repo를 평가한다 — setup friction, task discoverability, test fixtures, small examples, validation logs, restartability.
4. **GPU-runtime surrogate for SciML kernels:** custom JAX/CUDA kernel optimization에서 LLM forecast를 cheap prefilter로 쓰고 실제 GPU measurement는 calibrated uncertainty가 큰 후보에만 배정한다.
