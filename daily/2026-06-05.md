## Added Resources

### CFD / ROM / simulator diagnostics

- [History-aware adaptive reduced-order models via incremental singular value decomposition](https://arxiv.org/abs/2605.28684) — offline ROM이 online regime shift에서 망가지는 문제를 full-order correction snapshot + incremental SVD basis update로 완화합니다. Burgers, Sod shock tube, rotating detonation engine case까지 보여줘서 compressible-flow ROM의 “deployment 이후 업데이트” 참고축입니다. `#ROM` `#iSVD` `#compressible-flow` `#online-adaptation`
- [Semigroup Consistency as a Diagnostic for Learned Physics Simulators](https://arxiv.org/abs/2605.26324) — learned simulator가 직접 s+t step 예측과 s 후 t step composition을 일관되게 내는지 normalized semigroup error로 진단합니다. Heat/Burgers + ConvNet/FNO에서 rollout degradation과 양의 상관을 보이며, CFD surrogate 평가에 field RMSE 외 장기 동역학 gate를 추가해야 한다는 신호입니다. `#learned-simulator` `#diagnostics` `#rollout` `#FNO`
- [MARUT: An Exascale-Ready, GPU-Accelerated High-Order CFD Framework with AMR for High-Speed Flows and Finite-Rate Chemistry](https://arxiv.org/abs/2605.26388) — NVIDIA GPU-resident MPI + high-order spectral DG + SSP-RK + dynamic AMR로 subsonic~hypersonic, reactive nonequilibrium flows를 겨냥합니다. surrogate 논문은 많지만, 이런 solver/data backbone이 있어야 고신호 CFD-ML이 가능합니다. `#CFD` `#GPU` `#AMR` `#reactive-flow`

### CAD-domain PDE / CAD–CAE automation

- [Multi-patch neural solver with isogeometric mappings for partial differential equations on computer-aided design domains](https://doi.org/10.1007/s00366-026-02351-z) — NURBS 기반 multi-patch isogeometric map의 parametric space에서 patch-local neural networks로 PDE를 풉니다. CAD geometry를 mesh 전처리의 적이 아니라 solver coordinate system으로 활용하려는 방향이라 CAD-to-CAE 자동화와 잘 맞습니다. `#CAD-domain` `#IGA` `#PINN` `#PDE-solver`
- [LLM-enabled automation of CAD–structural analysis via Model Context Protocol](https://doi.org/10.1016/j.autcon.2026.107030) — Automation in Construction 논문으로, MCP를 통해 CAD와 structural analysis 자동화를 연결합니다. agentic CAE에서 MCP가 단순 tool demo가 아니라 CAD→analysis workflow interface로 논문화되기 시작했다는 점이 중요합니다. `#MCP` `#CAD` `#structural-analysis` `#CAE-automation`
- [onshape-cli](https://github.com/am-will/onshape-cli) — Onshape REST API를 Python/Node CLI와 machine-readable JSON contract로 노출해 agent가 document/sketch/extrude/fillet/assembly/export/measure를 반복 실행할 수 있게 합니다. CAD agent에는 browser control보다 deterministic CLI/API + validation/export command가 더 맞다는 좋은 구현 신호입니다. `#Onshape` `#CAD-CLI` `#agent-tools` `#parametric-CAD`

### MCP / agent-readable engineering tools

- [archicad-mcp](https://github.com/Boti-Ormandi/archicad-mcp) — Archicad + Tapir JSON API를 MCP로 연결하고, 173개 underlying command를 4개 high-level tools와 script-first `execute_script`로 다룹니다. tool 수를 무작정 늘리기보다 live schema/search + Python script layer로 복잡한 CAD/BIM workflow를 한 번에 처리하는 설계가 참고할 만합니다. `#Archicad` `#MCP` `#BIM` `#script-first`
- [FreeCad_MCP_Server](https://github.com/JakobThiessen/FreeCad_MCP_Server) — FreeCAD GUI instance와 Python FastMCP server를 XML-RPC로 연결해 Sketcher/Part/PartDesign, constraints, STEP/STL export, screenshots를 도구화합니다. 아직 초기지만 “running CAD app + local RPC + MCP” 패턴을 비교할 수 있는 또 하나의 reference입니다. `#FreeCAD` `#MCP` `#CAD-automation` `#parametric-modeling`
- [cadlens-mcp](https://github.com/cadlens-co/cadlens-mcp) — DWG/DXF/DWF/DGN/PDF CAD 파일을 CADLens API로 parse해 entity/layer/metadata JSON을 MCP client에 반환합니다. CAD agent가 수정하기 전에 기존 drawing을 구조적으로 읽는 layer가 필요하다는 점에서 direct CAD control MCP와 보완 관계입니다. `#CAD-parsing` `#MCP` `#DWG` `#engineering-context`

## Best Find of the Day

**Semigroup Consistency as a Diagnostic for Learned Physics Simulators**를 오늘의 best find로 봅니다. 이유는 간단합니다. CFD/thermal surrogate에서 “짧은 horizon RMSE가 낮다”는 말은 실제 deployment 안정성을 거의 보장하지 않습니다. 이 논문은 learned time-stepper가 solution-map composition을 지키는지 보는 model-agnostic post-hoc diagnostic을 제안합니다. VA식 validation cascade에 넣기 좋습니다: field RMSE → spectral error → conservation/BC check → semigroup consistency → rollout degradation.

## Possible Research Ideas

1. **CFD surrogate rollout contract:** FNO/Graph surrogate 결과에 semigroup error, spectral-band error, conservation residual, OOD sensor reconstruction error를 함께 기록하는 `surrogate_eval.json` schema를 만든다.
2. **Adaptive ROM vs neural surrogate 비교:** Burgers/Sod/open cavity case에서 iSVD adaptive ROM과 FNO/DeepONet을 같은 online regime-shift protocol로 비교한다.
3. **CAD-agent interface matrix:** Onshape CLI, FreeCAD MCP, Archicad MCP, CADLens MCP를 “create/edit/query/validate/export/provenance” 축으로 비교해 VA용 CAD-to-CAE 최소 interface를 정의한다.
4. **CAD-domain neural PDE smoke test:** multi-patch IGA neural solver 아이디어를 단순 NURBS patch heat conduction 문제로 재현하고, mesh-based FEM/PINN과 boundary conformity를 비교한다.
5. **Solver-backed design automation loop:** CAD 생성 → geometric validation → export → meshability check → cheap CFD/FEA smoke run → report artifact를 하나의 MCP/CLI workflow로 묶는다.
