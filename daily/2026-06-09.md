## Added Resources

### CFD / SciML surrogate / control reliability

- [Drifting Models for Surrogate Flow Modeling](https://arxiv.org/abs/2606.07481) — indoor-environment CFD flow field 생성을 위해 generative drifting을 VAE latent space에 적용합니다. Diffusion-style iterative sampling보다 빠른 single-pass generation을 목표로 하며, label-aware masking으로 boundary condition alignment를 다룹니다. Real-time CFD surrogate가 필요한 design exploration/digital twin 쪽에서 볼 가치가 큽니다. `#CFD-surrogate` `#generative-model` `#real-time` `#boundary-condition`
- [No-Harm Physics-Informed Inverse Learning with Residual-Calibrated Uncertainty](https://arxiv.org/abs/2606.07153) — PDE inverse problem에서 learned reconstruction이 residual-calibrated radius 기준으로 baseline보다 나쁘면 baseline을 반환하는 certification-and-selection layer를 제안합니다. “더 똑똑한 모델”보다 “나빠지지 않는 gate”를 먼저 둔다는 점이 thermal/CFD surrogate governance와 잘 맞습니다. `#physics-informed-learning` `#inverse-problem` `#uncertainty` `#validation-gate`
- [Investigating frictional instability due to pressurization in granular media: insights from coupled CFD-DEM](https://arxiv.org/abs/2606.06257) — pore-pressure-induced granular layer reactivation을 3D coupled CFD-DEM으로 분석합니다. Pure ML 논문은 아니지만 drainage, permeability proxy, fabric degradation 같은 hidden field가 instability를 좌우한다는 점에서 surrogate target 설계와 multi-physics validation에 참고됩니다. `#CFD-DEM` `#multiphysics` `#porous-media` `#validation-data`

### Thermal / heat-transfer constraints

- [On shifting the thermal explosion threshold by a vortical flow in dimension two](https://arxiv.org/abs/2606.07307) — Frank-Kamenetskii thermal explosion threshold가 2D vortical flow 방향/구조에 의해 조정될 수 있음을 이론적으로 다룹니다. ML 논문은 아니지만 heat-transfer/control 문제에서 flow field가 안전 임계값 자체를 바꾼다는 좋은 물리 anchor입니다. `#heat-transfer` `#thermal-safety` `#vortical-flow` `#combustion`
- [Space-CIM: Enabling Compute-In-Memory Accelerators for Thermally-Constrained Space Platforms](https://arxiv.org/abs/2606.05741) — orbital/space compute에서 radiator capacity를 loop 안에 넣어 GPU+HBM과 compute-in-memory accelerator의 TOPS/W를 비교합니다. CFD/thermal 설계 관점에서는 “AI workload도 결국 thermal boundary condition 안에서 성능이 정해진다”는 co-design 사례입니다. `#thermal-design` `#radiator` `#accelerator` `#co-design`

### CAD / MCP / agentic engineering workflow

- [GuideCAD: A Lightweight Multimodal Framework for 3D CAD Model Generation via Prefix Embedding](https://arxiv.org/abs/2606.07024) · [code](https://github.com/mskimS2/GuideCAD) — image embedding을 prefix embedding으로 바꿔 pretrained LLM + transformer decoder가 CAD construction sequence를 생성하게 합니다. Text/image-to-CAD에서 parameter-efficient training과 executable sequence가 함께 중요해지는 신호입니다. `#CAD` `#text-to-CAD` `#multimodal` `#construction-sequence`
- [sandraschi/qcad-mcp](https://github.com/sandraschi/qcad-mcp) — QCAD Pro architectural CAD를 FastMCP 3.2 server + webapp으로 노출하려는 초기 repo입니다. 3D FreeCAD뿐 아니라 2D drawing/CAD review도 agent-callable surface로 열리는 신호지만, adoption은 아직 낮아 실험 후보로만 보는 게 맞습니다. `#MCP` `#QCAD` `#CAD-automation` `#early-tool`
- [Codex Integration with Creative Industry Software](https://github.com/jianbaorui07-dot/Codex-Integration-with-Creative-Industry-Software) — Windows-first local MCP stdio server/safety bridge로 Blender, AutoCAD/DXF, Photoshop 등 creative/engineering-adjacent tools를 연결하려는 repo입니다. 핵심은 개별 앱보다 “local safety bridge + tool adapter” 패턴이며, CAD/CAE agent도 이 구조를 필요로 합니다. `#MCP` `#AutoCAD` `#DXF` `#agent-safety-bridge`

## Best Find of the Day

**Drifting Models for Surrogate Flow Modeling** 을 오늘의 best find로 봅니다. CFD surrogate가 실제 design exploration에 들어가려면 field quality만큼 inference latency가 중요합니다. 이 논문은 diffusion류 generative surrogate의 품질 장점을 유지하면서 single-pass 생성으로 속도 병목을 줄이려는 방향이라, real-time CFD surrogate / interactive design / indoor thermal-flow optimization에 직접 닿습니다.

## Possible Research Ideas

1. **Single-pass vs iterative surrogate benchmark:** 같은 CFD dataset에서 diffusion, consistency/distillation, drifting, deterministic FNO/POD-NN을 field error + boundary-condition consistency + latency로 비교한다.
2. **No-harm surrogate gate:** CFD/thermal inverse problem에서 learned output을 baseline POD/Kriging/solver fallback과 비교해 “나빠지면 baseline 반환” contract를 구현한다.
3. **CAD sequence validation smoke test:** GuideCAD/UniCAD류 construction sequence를 CadQuery/FreeCAD로 실행하고, export 가능성·watertightness·named boundary 보존 여부를 자동 체크한다.
4. **Thermal constraint-first design sheet:** accelerator/space radiator 사례처럼 model architecture 전에 heat source, cooling boundary, allowable temperature, safety margin을 먼저 적는 template을 만든다.
