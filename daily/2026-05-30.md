# Daily Research Radar — 2026-05-30

## Highlights

- **오늘의 sciML 신호는 “surrogate를 실제 시스템·제어·검증 workflow로 내리는 법”입니다.** SMR steam-generator CFD surrogate, BuilDyn thermal-control data generation, multifidelity POD가 모두 단순 benchmark보다 operating envelope와 offline simulation budget을 묻고 있습니다.
- **CFD surrogate의 난점은 Reynolds/geometry/domain shift로 계속 수렴합니다.** Cross-Reynolds generalization 논문은 FNO가 10x Reynolds shift에서 크게 흔들리고, representation retrieval baseline이 의외로 강하다는 점을 보여줘 “모델 크기”보다 latent geometry 진단이 중요합니다.
- **전통 ROM과 neural operator가 다시 만나는 흐름이 강합니다.** SMR HCSG 논문은 AE/CAE ROM + DeepONet/FNO를 비교하고, multifidelity POD는 고충실도 snapshot 비용을 낮추는 수학적 route를 제시합니다. VA의 CFD surrogate asset 설계에 바로 연결됩니다.
- **HPC solver 쪽 best tool signal은 GALÆXI입니다.** vendor-agnostic GPU DGSEM open-source toolchain은 NVIDIA/AMD 혼재 환경에서 high-order compressible CFD를 돌리려는 흐름이라, “AI surrogate 이전의 신뢰 가능한 데이터 생산” 축으로 중요합니다.
- **Agentic engineering의 교훈은 꽤 차갑습니다.** CLAX-PT case study는 AI coding agent가 oracle test를 통과해도 물리적으로 틀린 correction을 만들 수 있음을 보여줍니다. CAD/CFD 자동화 agent에도 “테스트 + 물리 개념 + changelog + 금지 규칙”이 필요합니다.

## Added Resources

### CFD / thermal-fluid surrogates & reduced models

- [Neural Operator-Based Surrogate Model for CFD: Helical Coil Steam Generator in Small Modular Reactor](https://arxiv.org/abs/2605.30277) — SMART SMR의 helical coil steam generator transient CFD를 digital twin에 쓰기 위해 ROM + neural operator를 결합합니다. unstructured/structured CFD data에 AE/CAE ROM을 나눠 쓰고 L-DeepONet/FNO를 비교하며, Kármán vortex street 예측을 위해 multi-scale 기법을 넣은 점이 실무형 surrogate에 가깝습니다. `#CFD` `#thermal-hydraulics` `#SMR` `#DeepONet` `#FNO`
- [Multifidelity Proper Orthogonal Decomposition](https://arxiv.org/abs/2605.29213) — 비싼 high-fidelity snapshot만으로 POD basis를 만들 때 생기는 비용/overfitting 문제를 low-fidelity sample과 control-variate weighting으로 줄이는 ROM 논문입니다. CFD/열전달 surrogate의 offline data budget을 줄이는 고전적이지만 강한 축입니다. `#POD` `#ROM` `#multifidelity` `#simulation-budget`
- [Striding Across Reynolds Numbers: Representation Geometry in Neural PDE Generalisation](https://arxiv.org/abs/2605.30112) — 2D Navier-Stokes에서 Reynolds number가 10배 바뀌면 FNO error가 크게 증가하고, representation matching/retrieval baseline이 의외로 경쟁력 있음을 보입니다. Cross-Re generalization을 모델 점수 하나가 아니라 latent representation geometry로 진단해야 한다는 신호입니다. `#Navier-Stokes` `#FNO` `#OOD` `#representation-geometry`
- [Deep Adaptive Dimension Reduction for Bayesian Inference in Inverse Problems](https://arxiv.org/abs/2605.29373) — PDE-governed inverse problem에서 Variational Flow와 adaptive dimension reduction을 결합해 고차원 posterior를 다룹니다. CFD/열전달 inverse design에서 surrogate가 mean prediction만 맞추는 대신 uncertainty/posterior 구조를 보존해야 할 때 참고할 만합니다. `#inverse-problem` `#Bayesian` `#dimension-reduction` `#PDE`

### Thermal dynamics & control data generation

- [BuilDyn: Excitation-Driven Data Generation for Building Thermal Dynamics Modeling and Control](https://arxiv.org/abs/2605.29849) — building thermal dynamics/control용 ML dataset이 fixed policy의 정상 운전 데이터에 갇히는 문제를 excitation-driven data generation으로 풀려는 Python package입니다. thermal digital twin이나 MPC surrogate는 “예쁜 historical data”보다 control state space를 충분히 흔든 데이터가 필요하다는 좋은 실무 신호입니다. `#thermal-control` `#building-energy` `#data-generation` `#MPC`

### Solver infrastructure & HPC CFD

- [An Architecture-Agnostic High-Order Discontinuous Galerkin Framework for Compressible Flows](https://arxiv.org/abs/2605.28627) — GALÆXI라는 open-source, architecture-agnostic DGSEM framework를 소개합니다. NVIDIA CUDA와 AMD HIP를 모두 지원하는 high-order compressible CFD toolchain이라는 점에서, surrogate training data를 만드는 고충실도 solver infrastructure로 볼 가치가 큽니다. `#CFD` `#DGSEM` `#GPU` `#HPC` `#compressible-flow`

### SciML diagnostics & agentic scientific software

- [Unveiling Multi-regime Patterns in SciML: Distinct Failure Modes and Regime-specific Optimization](https://arxiv.org/abs/2605.29153) — SciML training dynamics가 hyperparameter에 따라 여러 regime과 failure mode로 갈라질 수 있음을 loss landscape/optimization 관점에서 분석합니다. PINN·neural operator 실험에서 “seed 하나 잘 나온 plot”보다 regime-aware diagnostics가 필요하다는 근거로 좋습니다. `#SciML` `#optimization` `#failure-modes` `#diagnostics`
- [Physics Is All You Need? A Case Study in Physicist-Supervised AI Development of Scientific Software](https://arxiv.org/abs/2605.30353) — Claude Code로 JAX 기반 scientific software(CLAX-PT)를 만든 12일/57세션 case study입니다. agent가 oracle tests를 통과해도 물리적으로 무의미한 correction을 만들 수 있었고, diverse parameter tests·shared changelog·unphysical patch 금지 규칙이 핵심 방어선이었다는 점이 CAD/CFD agent workflow에 그대로 적용됩니다. `#agentic-workflow` `#scientific-software` `#JAX` `#verification`

## Best Find of the Day

**SMR helical-coil steam-generator neural-operator surrogate**를 오늘의 best find로 봅니다. 이유는 단순한 FNO benchmark가 아니라, 복잡한 열수력 장치의 transient CFD를 digital twin에 넣기 위해 structured/unstructured data ROM, DeepONet/FNO 비교, multi-scale vortex 예측을 한 workflow이기 때문입니다. VA가 관심 있는 CAD/CFD surrogate도 결국 “복잡 형상 + transient field + 검증 가능한 operating envelope” 문제라서 읽을 가치가 큽니다.

## Possible Research Ideas

1. **Surrogate asset contract for thermal-fluid DT:** geometry/mesh type, ROM basis, neural operator weights, Reynolds/operating envelope, vortex/thermal KPI validation plot을 하나의 artifact schema로 묶는다.
2. **Cross-Re representation audit:** FNO/GNO/autoencoder surrogate의 latent space에서 Reynolds shift가 어떻게 보이는지 retrieval baseline과 같이 비교한다.
3. **Excitation-aware thermal dataset generator:** BuilDyn 아이디어를 2D heat exchanger나 electronics-cooling toy problem에 옮겨, fixed-policy data vs actively excited data의 MPC 성능 차이를 본다.
4. **Multifidelity POD before neural operator:** low-fidelity CFD/ROM sample로 POD space를 먼저 안정화한 뒤, residual이나 latent dynamics만 neural operator로 학습한다.
5. **Agentic CFD guardrail template:** CLAX-PT case study를 참고해 “oracle tests 통과 + diverse parameter tests + physical no-fudge rules + changelog review”를 OpenFOAM/CAD automation agent에 강제한다.
