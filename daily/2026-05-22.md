# Daily Research Radar — 2026-05-22

## Highlights

- **오늘의 가장 강한 신호는 “engineering surrogate를 benchmark와 closed loop 안에서 평가하자”입니다.** ShapeBench와 COSMO-Agent는 CAD/CAE/design optimization이 단순 생성물이 아니라 검증 가능한 반복 루프가 되어야 한다는 방향을 보여줍니다.
- **Neural operator 쪽은 data efficiency와 sample selection이 다시 핵심입니다.** Physics-based active learning은 expensive PDE/CFD 데이터셋을 무작정 늘리는 대신 “어떤 simulation을 추가로 돌릴지”를 고르는 문제로 이동합니다.
- **CAD agent 평가는 더 구체화되고 있습니다.** Text2CAD-Bench와 COSMO-Agent는 text-to-CAD/optimization loop를 parametric geometry, simulation feedback, valid edit로 묶어 평가하려는 신호입니다.
- **MCP/tool ecosystem은 안전한 tool-call 데이터와 실제 API 검증으로 이동 중입니다.** Firefly는 real MCP servers에서 verified tool-call data를 만들려는 흐름이라 engineering agents의 평가/훈련 데이터 설계에 유용합니다.

## Added Resources

### SciML, CFD surrogate & optimization

- [ShapeBench](https://arxiv.org/abs/2605.20763) — aerodynamic shape optimization benchmark/diagnostic suite. 각 task에 fast surrogate를 제공하고 가능한 경우 high-fidelity CFD pipeline으로 최종 검증해 fidelity gap을 분석하려는 점이 좋습니다. “AI surrogate로 찾은 설계가 실제 CFD에서도 좋은가?”를 묻는 benchmark 신호입니다. `#CFD` `#aerodynamics` `#shape-optimization` `#benchmark`
- [Data-Efficient Neural Operator Training via Physics-Based Active Learning](https://arxiv.org/abs/2605.21348) — neural operator 학습 데이터가 비싼 상황에서 physics-based active learning으로 informative PDE samples를 선택합니다. CFD/열전달 surrogate를 만들 때 “dataset size”보다 “simulation acquisition policy”가 중요해지는 방향입니다. `#neural-operator` `#active-learning` `#PDE` `#data-efficiency`
- [Hierarchical Multi-Fidelity Learning for Predicting 3D Flame Wrinkling and Turbulent Burning Velocity](https://arxiv.org/abs/2605.08232) — combustion/turbulent flame에서 high/low fidelity data를 계층적으로 묶는 SciML 응용. 열·유동·반응이 결합된 문제에서 data-limited surrogate를 만드는 실용 예제로 볼 만합니다. `#combustion` `#multi-fidelity` `#turbulence` `#SciML`
- [ShapeBench + HiLiftAeroML 연결 관찰](https://arxiv.org/abs/2605.19565) — 어제의 HiLiftAeroML과 오늘 ShapeBench를 같이 보면 CFD-AI가 “데이터셋 공개”에서 “optimization protocol + final CFD verification”로 넘어가는 흐름이 보입니다. `#CFD-dataset` `#verification` `#surrogate-benchmark`

### CAD, CAE & agentic engineering workflow

- [COSMO-Agent](https://arxiv.org/abs/2605.20190) — CAD-CAE semantic gap을 closed-loop optimization/simulation/modeling orchestration 문제로 다루는 tool-augmented RL framework. simulation feedback을 valid geometry edit로 바꾸는 병목을 정면으로 잡아서 CAD→CAE 자동화 관점에서 high-signal입니다. `#CAD-CAE` `#agentic-engineering` `#optimization` `#simulation-loop`
- [Text2CAD-Bench](https://arxiv.org/abs/2605.18430) — text-to-parametric-CAD benchmark. primitive sketch/extrude를 넘어 복잡 topology, freeform surface, procedural prompt style을 포함해 CAD agent를 더 빡세게 재는 방향입니다. `#CAD` `#benchmark` `#parametric-CAD` `#text-to-CAD`
- [mcp-cad](https://github.com/PaulGregoryBaker/mcp-cad) — AI-driven sheet-metal manufacturing orchestrator를 위한 MCP layer. 아직 stars/release는 없지만 sheet metal처럼 제조 제약이 강한 도메인에서 agent-control surface를 만들려는 신호라 추적 가치가 있습니다. `#MCP` `#sheet-metal` `#manufacturing` `#agent-tools`
- [Firefly](https://arxiv.org/abs/2605.17558) — real MCP servers를 탐색해 verified tool-call data를 생성하는 pipeline. engineering agents를 학습/평가할 때 mock task가 아니라 실제 API behavior와 ground-truth outcome을 묶어야 한다는 점에서 중요합니다. `#MCP` `#tool-calling` `#agent-evaluation` `#verified-data`

## Best Find of the Day

**ShapeBench**가 오늘의 best research find입니다. CFD surrogate가 설계 최적화에 들어가려면 단순 field error보다 “surrogate가 제안한 optimum이 high-fidelity CFD에서 살아남는가”가 훨씬 중요합니다. ShapeBench는 fast surrogate와 final CFD verification을 같이 두려는 점에서 VA의 CAD/CFD design loop 평가 기준으로 쓸 만합니다.

workflow 쪽에서는 **COSMO-Agent**가 중요합니다. CAD agent의 진짜 병목은 “그럴듯한 모델 생성”보다 simulation feedback을 valid, editable, constraint-satisfying geometry change로 번역하는 일입니다. 이건 CAD-CAE 자동화의 핵심 관문입니다.

## Possible Research Ideas

1. **Surrogate optimization audit:** ShapeBench식으로 surrogate optimum → high-fidelity CFD recheck → fidelity-gap report를 기본 artifact로 강제하는 작은 template를 만든다.
2. **Active-learning CFD budgeter:** neural operator 학습용 CFD samples를 random/LHS가 아니라 physics-based acquisition으로 고르는 mini experiment를 HiLiftAeroML 또는 공개 airfoil dataset에서 돌린다.
3. **CAD-CAE edit contract:** COSMO-Agent류 workflow를 참고해 simulation feedback을 geometry edit로 바꿀 때 필요한 schema를 정의한다: editable parameter, constraint, affected faces, expected physics effect, validation test.
