# Daily Research Radar — 2026-05-27

## Highlights

- **오늘의 핵심 신호는 “surrogate를 uncertainty-aware inverse design/control loop 안에 넣기”입니다.** Bayesian inverse design 논문은 neural operator를 MCMC loop에 직접 넣어 posterior 구조를 유지하면서 1000× 이상 가속하는 방향을 보여줍니다.
- **thermal/CFD surrogate는 복잡 geometry와 boundary condition을 더 정면으로 다루고 있습니다.** NPSolver는 irregular domain Poisson problem에서 PCG step을 supervision으로 쓰고, complex terrain wind-field neural operator는 sparse observation까지 결합합니다.
- **고신호 CFD 데이터는 여전히 domain-specific입니다.** PWR full-scale flow-field dataset은 원전 lower-plenum/core-inlet의 swirl·mixing physics를 ML reconstruction/prediction task로 연결합니다.
- **PINN 계열은 “loss weighting”보다 representation/derivative 계산 효율 쪽 개선이 더 흥미롭습니다.** Fourier feature pyramid PINN(beignet)은 bandlimit 제어와 FFT 기반 derivative로 어려운 PDE residual minimization을 밀어붙입니다.
- **CAD/3D agent 쪽은 visual plausibility에서 operability contract로 이동 중입니다.** Hylos와 memory-augmented CAD RL agent 모두 “생긴 것”보다 entity/frame/constraint/action/verification이 살아있는 3D를 강조합니다.

## Added Resources

### CFD inverse design, thermal control & neural operators

- [Accelerating Bayesian inverse design in computational fluid dynamics using neural operators](https://arxiv.org/abs/2605.26059) — neural operator surrogate를 Bayesian MCMC inverse-design loop 안에 넣고, quasi-1D nozzle geometry posterior/uncertainty trend를 CFD reference와 비교합니다. sparse flow observation → geometry inference를 uncertainty-aware하게 돌리는 방향이라 aerodynamic inverse design과 surrogate validation 논의에 좋습니다. `#CFD` `#inverse-design` `#neural-operator` `#uncertainty`
- [NPSolver: Neural Poisson Solver with Iterative Physics Supervision](https://arxiv.org/abs/2605.25786) · [code](https://github.com/intell-sci-comput/NPSolver) — solution labels나 raw residual만 쓰지 않고, 소수 PCG step으로 자기 예측을 refine한 결과를 supervision signal로 쓰는 neural Poisson solver. irregular 2D/3D geometry와 mixed boundary condition, downstream thermal boundary control까지 다뤄 heat-transfer/control surrogate에 실용적인 힌트를 줍니다. `#Poisson` `#thermal-control` `#physics-supervision` `#irregular-domain`
- [Transformer-based Neural Operators for 3D Wind Field Prediction over Complex Mountainous Terrain](https://arxiv.org/abs/2605.25679) — mesh generation과 solver cost가 큰 complex terrain wind field를 point/graph neural-operator instantiation으로 다루고, real mountainous site zero-shot 및 1% sparse observation assimilation을 평가합니다. terrain/geometry가 강한 CFD surrogate에서 “관측 결합형 operator” 방향이 고신호입니다. `#wind-field` `#complex-terrain` `#neural-operator` `#sparse-observation`
- [Fourier Feature Pyramids for Physics-Informed Neural Networks](https://arxiv.org/abs/2605.24278) — trainable multi-resolution Fourier feature pyramid(beignet)로 PINN representation을 바꾸고, spectral derivative/controlled bandlimit로 어려운 PDE residual optimization을 개선합니다. PINN을 쓸 때 architecture scaling보다 representation/derivative 계산을 먼저 봐야 한다는 신호입니다. `#PINN` `#Fourier-features` `#PDE-solver` `#neural-field`

### CFD datasets, reconstruction & domain-specific surrogates

- [High-fidelity Modeling of Full-scale Pressurized Water Reactor Flow Fields for Machine Learning Applications](https://arxiv.org/abs/2605.24763) — 공개 geometry/operating condition 기반 PWR lower-plenum/core-inlet CFD를 만들고, partial field reconstruction과 short-term autoregressive prediction을 평가합니다. spatially aware ConvLSTM이 DeepONet/LSTM보다 나았다는 결과는 domain physics와 sensor layout이 surrogate architecture 선택을 좌우한다는 reminder입니다. `#CFD-dataset` `#nuclear` `#field-reconstruction` `#ConvLSTM`
- [Physical Fidelity Reconstruction via Improved Consistency-Distilled Flow Matching for Dynamical Systems](https://arxiv.org/abs/2605.05975) — flow-matching teacher를 one-step consistency model로 distill해 smoke buoyancy, turbulent channel, Kolmogorov flow의 coarse-to-fine reconstruction을 빠르게 수행합니다. generative fluid model을 실시간 visualization/simulation-in-the-loop로 가져오려면 distillation이 핵심일 수 있습니다. `#flow-matching` `#fluid-reconstruction` `#distillation` `#real-time`

### CAD, 3D operability & intelligent engineering agents

- [Hylos: Operability Contracts for Model-Native Spatial Intelligence](https://arxiv.org/abs/2605.24728) — 3D 생성물이 실제 CAD/robotics/simulation substrate가 되려면 entity, frame, surface anchor, admissible action, effect diff, rollback/review boundary가 필요하다는 systems preprint. VA식 CAD→mesh→CFD agent workflow에서 “operable 3D contract”를 설계할 때 좋은 언어를 줍니다. `#spatial-AI` `#operability` `#CAD-agent` `#validation-contract`
- [Memory-Augmented Reinforcement Learning Agent for CAD Generation](https://arxiv.org/abs/2605.19748) — geometric kernel을 structured toolchain으로 감싸고, case/skill memory + RL retrieval/policy optimization으로 복잡 CAD operation sequence의 성공률과 geometric consistency를 높이려는 접근. CAD agent가 예제 retrieval trap에 빠지지 않도록 memory를 어떻게 써야 하는지 참고할 만합니다. `#CAD-generation` `#RL-agent` `#toolchain` `#design-automation`

## Best Find of the Day

**NPSolver**를 오늘의 best find로 봅니다. 이유는 단순한 neural operator paper가 아니라, classical iterative solver의 일부 PCG step을 학습 supervision으로 재사용하는 방식이기 때문입니다. 이는 “solver를 버리고 ML로 대체”가 아니라 “검증 가능한 수치해석 절차를 학습 신호/제어 loop 안에 넣기”에 가깝고, VA 관심사인 heat-transfer boundary control, irregular CAD domain, solver-assist surrogate와 잘 맞습니다.

## Possible Research Ideas

1. **Poisson/thermal mini-benchmark:** irregular 2D heat-conduction domain에서 classical solver, PINN, NPSolver-style iterative supervision을 비교하고 KPI를 field error + boundary-control objective + wall-clock으로 둔다.
2. **Sparse-sensor CFD surrogate test:** terrain wind 논문 패턴처럼 OpenFOAM duct/heat-sink case에서 1–5% sparse sensors를 넣었을 때 neural operator reconstruction이 얼마나 안정화되는지 본다.
3. **Operable CAD contract schema:** Hylos 관점으로 CAD artifact에 `entities`, `frames`, `constraints`, `admissible_actions`, `validation_failures`, `rollback_plan`을 붙이고 mesh/export smoke test와 연결한다.
4. **Distilled flow reconstruction demo:** low-res CFD/thermal field → high-res field reconstruction에서 flow-matching teacher와 one-step distilled student를 비교하되, spectrum/gradient/conservation metric을 필수로 둔다.
