# Daily Research Radar — 2026-05-20

## Highlights

- **Neural-operator reliability is moving from “bigger FNO” to physics-aligned invariance and long-horizon diagnostics.** PACE-FNO uses known symmetries to canonicalize fields before prediction, while CTA-Swin-UNet + resolvent reconstruction targets long-rollout wall turbulence.
- **Thermal/heat-transfer ML is giving useful deployment patterns, not just toy PDEs.** In-context subsurface temperature prediction and data-center cooling digital twins both connect sparse observations, uncertainty/surrogates, and operational constraints.
- **CAD automation benchmarks are getting more engineering-native.** Text2CAD-Bench stresses complex topology/freeform/application diversity, and FEA-feedback CAD agents show why visual plausibility is nowhere near enough.
- **Agentic engineering workflows need testable harnesses.** Code-as-agent-harness, PROTEA, and GraphMind are not CFD-specific, but they point toward traceable workflow graphs, offline evaluation, and executable verification for CAD/CAE agents.
- **Open-source tools worth tracking today are practical infrastructure:** pyforce for VTK-friendly reduced-order modeling, plus Onshape/KiCAD/FreeCAD MCP bridges as evidence that CAD tools are becoming agent-addressable.

## Added Resources

### SciML, CFD surrogate reliability & thermal systems

- [Physics-Aligned Canonical Equivariant Fourier Neural Operator under Symmetry-Induced Shifts](https://arxiv.org/abs/2605.18606) — PACE-FNO estimates a symmetry frame, maps fields into a canonical reference frame, applies FNO, then transforms predictions back. Important because CFD/thermal operators often have translation/rotation/frame symmetries that should be enforced rather than relearned. `#neural-operator` `#equivariance` `#OOD-generalization` `#SciML`
- [Long-horizon prediction of three-dimensional wall-bounded turbulence with CTA-Swin-UNet and resolvent analysis](https://arxiv.org/abs/2605.17888) — combines channel-time-attention Swin-UNet for planar prediction with resolvent-based spectral linear stochastic estimation to reconstruct 3D wall-bounded turbulence. Useful signal for surrogate workflows where long autoregressive stability matters more than one-step MSE. `#turbulence` `#long-horizon` `#surrogate` `#resolvent-analysis`
- [In-context learning enables continental-scale subsurface temperature prediction from sparse local observations](https://arxiv.org/abs/2605.16665) — transformer-based In-Context Earth predicts temperature-at-depth fields from sparse borehole observations with calibrated uncertainty. Relevant to thermal-field inference, geothermal assessment, and sparse-sensor heat-transfer problems. `#thermal` `#geothermal` `#in-context-learning` `#uncertainty`
- [Co-Design Optimization for Data Center Cooling System via Digital Twin](https://arxiv.org/abs/2605.15516) — reduced-order surrogate + Modelica digital twin evaluates coolant distribution units, flow allocation, and supply temperature strategies for Frontier-scale liquid cooling. Good engineering example of surrogate-assisted thermal control under safety constraints. `#thermal-control` `#digital-twin` `#cooling` `#optimization`
- [pyforce-1.0.0](https://arxiv.org/abs/2605.18082) — Python framework for data-driven reduced-order modeling of multi-physics problems, rewritten around PyVista/VTK-compatible arrays. Practical because VTK export compatibility makes it easier to attach ROM/SciML workflows to existing CFD/CAE solvers. `#ROM` `#multi-physics` `#PyVista` `#VTK`

### CAD, MCP & engineering agents

- [Text2CAD-Bench](https://arxiv.org/abs/2605.18430) — 600 human-curated text-to-parametric-CAD examples across basic geometry, complex topology, freeform surfaces, and non-mechanical domains with both non-expert and procedural prompt styles. Good evaluation pressure beyond primitive sketch-extrude tasks. `#CAD` `#benchmark` `#parametric-CAD` `#text-to-CAD`
- [Self-Improving CAD Generation Agents with Finite Element Analysis as Feedback](https://arxiv.org/abs/2605.17448) — asks agents to produce assembled multi-part STEP files from engineering briefs and validates them with FEA; first-attempt strict pass rates are effectively zero. Strong reminder that CAD agents need simulation feedback and repair loops, not just shape generation. `#CAD-agent` `#FEA` `#STEP` `#validation`
- [Code as Agent Harness](https://arxiv.org/abs/2605.18747) — survey/framing paper arguing that code is becoming the operational substrate for agent reasoning, action, environment modeling, and execution-based verification. Useful lens for CAD/CAE agents: harness code should expose solver state, artifacts, checks, and provenance. `#agent-harness` `#verification` `#workflow`
- [PROTEA](https://arxiv.org/abs/2605.18032) — offline, test-driven evaluation and iterative refinement interface for multi-agent LLM workflows. Relevant for engineering automation because multi-step CAD→mesh→solve→postprocess workflows need node-level failure localization, not only final-answer judgment. `#multi-agent` `#evaluation` `#workflow-debugging`
- [Jarvis Onshape MCP](https://github.com/ReshefElisha/jarvis-onshape-mcp) — Onshape MCP plugin for driving real CAD with vision decomposition, truth-telling, parametric iteration, and FeatureScript. Worth tracking as a cloud-CAD counterpart to FreeCAD/CadQuery MCP workflows. `#MCP` `#Onshape` `#CAD-automation` `#FeatureScript`

## Best Find of the Day

**PACE-FNO** is the best research find; **Self-Improving CAD Agents with FEA feedback** is the best workflow warning. Together they say the same thing in two domains: engineering ML should encode the invariants and checks the physics already gives us. For VA's CAD→thermal/CFD automation, that means symmetry/structure-aware surrogates plus artifact gates: STEP validity, named faces, meshability, boundary-condition tags, solver logs, and uncertainty/error diagnostics.

## Possible Research Ideas

1. **Symmetry sanity benchmark:** take a simple periodic convection/diffusion or channel-flow slice dataset and compare vanilla FNO vs symmetry-canonicalized preprocessing under shifted/rotated inputs.
2. **Thermal digital-twin mini case:** build a reduced-order cooling-loop toy model where the controller optimizes flow rate and inlet temperature under max-temperature constraints; compare surrogate error vs safety margin.
3. **CAD-to-FEA/CFD validation contract:** define a small STEP-generation task where agents must pass dimensional checks, assembly/contact checks, mesh generation, and one solver smoke test before the artifact is accepted.
