## New Papers / Tools

### Roshera-CAD

- Link: https://github.com/varun29ankuS/Roshera-CAD
- Type: CAD / geometry kernel / MCP bridge
- Why it matters: Roshera puts the trust boundary in the kernel: every operation is expected to return a validity certificate, so CAD agents can query/build/verify mechanical geometry instead of only generating plausible shapes.
- VA relevance: Strong reference for Urban_Flighter-style CAD/CAE agents: geometry edits should leave B-Rep validity, watertight/manifold checks, and provenance artifacts.
- Caveat: Fair Source / FSL-1.1-Apache-2.0, not plain permissive OSS today. README claims need local build/test before trusting certificate semantics or performance.
- Action: Watch/Test

### DSGNAR — well-conditioned PINN training

- Link: https://arxiv.org/abs/2607.02194
- Type: Paper / PINN optimizer
- Why it matters: The paper treats PINN underperformance as ill-conditioned optimization rather than just weak architecture. It reports large accuracy gains across nonlinear, chaotic, multi-scale, high-dimensional, and Navier-Stokes problems.
- VA relevance: Good citation for the argument that solver-grade SciML needs numerical conditioning and acceptance tests, not only neural operator branding.
- Caveat: Code/repro harness still needs checking; do not treat reported precision claims as industrial CFD validation.
- Action: Cite

### Embodied.cpp

- Link: https://github.com/SEU-PAISys/Embodied.cpp
- Paper: https://arxiv.org/abs/2607.02501
- Type: Embodied AI runtime / deployment infrastructure
- Why it matters: It packages VLA/WAM deployment around adapters, sequence builders, backbone execution, head plugins, serving, GGUF conversion, and simulator evaluation clients.
- VA relevance: Pattern transfers to hardtech/CAE agents: runtime contracts and typed simulator/robot adapters matter more than a pretty chat surface.
- Caveat: Robotics-first; useful as infrastructure pattern, not directly CFD/CAD.
- Action: Save
