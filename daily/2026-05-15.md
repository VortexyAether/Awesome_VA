# Daily Research Radar — 2026-05-15

## Highlights

- **CFD surrogate 쪽 핵심 신호는 “큰 모델”보다 routing, composition, coordinate registration이다.** Shodh-MoE, PDE expert composition, shock-centered micro-nozzle가 모두 dense universal surrogate의 한계를 다른 각도에서 찌른다.
- **Sharp/localized physics는 아직 neural operator의 약점이다.** U-HNO는 shocks/interfaces/local high-frequency content를 sparse routing으로 다루고, micro-nozzle 논문은 shock-centered coordinate만으로도 low-rank structure가 훨씬 잘 드러난다고 본다.
- **CAD/제조 ML은 geometry만으로 부족하고 engineering semantics가 필요하다.** Sheet-metal bending effort 논문은 bend intent/surface role 같은 feature가 manufacturability prediction에 중요하다는 근거다.
- **Agentic engineering workflow의 병목은 tool calling 자체가 아니라 검증 가능한 tool surface다.** CADCLAW, agent-ready OpenAPI, SPIN은 CAD/CAE agent가 evidence, schema, plan contract를 가져야 함을 보여준다.
- **Design/control 응용은 surrogate-in-the-loop로 읽어야 한다.** Turbine-blade PINN, reduced shape-design embedding, explainable MPC는 “예측 모델”보다 screening/control/optimization loop 안에서의 신뢰성이 관건이다.

## Added Resources

### Neural operators, CFD surrogates & SciML foundations

- [Eradicating Negative Transfer in Multi-Physics Foundation Models via Sparse Mixture-of-Experts Routing](https://arxiv.org/abs/2605.15179) — multi-physics co-training에서 open-channel flow, porous media처럼 incompatible regime이 dense neural operator에 negative transfer를 만든다는 문제를 sparse MoE routing으로 다룬다. `#SciML` `#multi-physics` `#MoE` `#neural-operator`
- [Discovering Physical Directions in Weight Space: Composing Neural PDE Experts](https://arxiv.org/abs/2605.14546) — fine-tuned PDE expert의 weight update를 physical parameter direction처럼 합성하려는 접근. Reynolds/geometry/material regime별 surrogate를 controllable family로 만드는 데 힌트가 있다. `#PDE-surrogate` `#expert-composition` `#transfer-learning`
- [U-HNO: A U-shaped Hybrid Neural Operator with Sparse-Point Adaptive Routing for Non-stationary PDE Dynamics](https://arxiv.org/abs/2605.12965) — smooth transport와 localized shock/interface/high-frequency feature가 공존하는 PDE를 hybrid operator + sparse routing으로 다룬다. `#neural-operator` `#shock` `#nonstationary-PDE`
- [Shock-Centered Low-Rank Structure and Neural-Operator Representation of Rarefied Micro-Nozzle Flows](https://arxiv.org/abs/2605.12723) — DSMC micro-nozzle flow를 shock location/thickness 기준으로 register하면 apparent complexity가 크게 줄어든다는 논문. architecture 전에 physics-aware coordinate가 중요하다는 좋은 사례. `#CFD` `#rarefied-flow` `#low-rank` `#neural-operator`
- [EqOD: Symmetry-Informed Stability Selection for PDE Identification](https://arxiv.org/abs/2605.11524) — Galilean invariance 같은 symmetry로 PDE discovery candidate를 줄여 noisy flow/transport data의 false positive를 억제한다. `#PDE-discovery` `#symmetry` `#SciML`

### CAD, manufacturability & design automation

- [CAD-feature enhanced machine learning for manufacturing effort estimation on sheet metal bending parts](https://arxiv.org/abs/2605.12266) — B-rep graph에 bend intent/surface role 같은 CAD feature semantics를 더해 sheet-metal manufacturability estimation을 개선한다. CAD-to-CAE도 named faces/BC intent 같은 semantics가 필요하다는 근거. `#CAD` `#manufacturing` `#B-rep` `#graph-ML`
- [A nonlinear extension of parametric model embedding for dimensionality reduction in parametric shape design](https://arxiv.org/abs/2605.11759) — high-dimensional shape parameter를 nonlinear reduced variables로 줄이면서 original design space로 backmapping한다. blade/duct/heat-exchanger optimization surrogate에 참고 가능. `#shape-optimization` `#design-automation` `#dimensionality-reduction`
- [A fast Physics-Informed Neural Networks based approach to the 2D design of turbine blades](https://arxiv.org/abs/2605.07131) — turbine blade preliminary design에서 progressive Euler-PINN과 geometry-aware loss weighting을 사용한다. thermal-fluid design screening anchor로 좋지만 off-design validation은 조심해야 한다. `#turbomachinery` `#PINN` `#design-screening`

### Control, MCP & agentic engineering workflows

- [CADCLAW](https://github.com/sunnyday-technologies/CADCLAW) — STEP assembly validation suite. inventory/interference/adjacency/dimension/orientation/floating-part/BOM/tolerance/render checks와 MCP exposure가 있어 “AI CAD → evidence-producing validation → CAE handoff” 패턴에 잘 맞는다. `#CAD` `#STEP` `#validation` `#MCP`
- [Making OpenAPI Documentation Agent-Ready](https://arxiv.org/abs/2605.14312) — stable REST API도 MCP/agent tool로 노출하면 documentation/schema/parameter semantics 부족 때문에 실패한다는 산업 사례. OpenFOAM/CAD helper API 설계에 바로 유용하다. `#MCP` `#agent-tools` `#API-quality`
- [SPIN: Structural LLM Planning via Iterative Navigation for Industrial Tasks](https://arxiv.org/abs/2605.14051) — validated DAG planning과 prefix execution control을 industrial task agent에 적용한다. CAD→mesh→solver처럼 비싼 workflow에서 “계획 검증 후 필요한 만큼만 실행”하는 패턴으로 쓸 만하다. `#industrial-agents` `#planning` `#workflow-validation`
- [FermiLink](https://github.com/TaoELi/FermiLink) — goal.md 기반 scientific simulation/HPC loop와 package knowledge base를 제공하는 agent framework. multi-day CFD/SciML campaign harness 관점에서 추적 가치가 있다. `#scientific-agents` `#HPC` `#simulation-workflow`

## Best Find of the Day

**CADCLAW**. 오늘 논문 신호도 좋지만, VA workflow 관점에서는 CADCLAW의 interface shape가 더 실용적이다. CAD/CAE agent는 geometry를 “그럴듯하게 만들기”보다 STEP/BOM/interference/tolerance/meshability evidence를 남겨야 한다. 이게 OpenFOAM/thermal simulation으로 넘어가기 전 missing contract layer에 가깝다.

## Possible Research Ideas

1. **CAD-to-CAE validation harness:** CadQuery/build123d generated geometry → CADTests-style unit tests → CADCLAW STEP assembly checks → minimal OpenFOAM meshability gate를 하나의 pipeline으로 묶는다.
2. **Regime-routed neural-operator benchmark:** heat equation + lid-driven cavity + shock/rarefied toy cases에서 dense FNO/JAX baseline과 MoE/routed operator를 비교해 negative transfer를 직접 측정한다.
3. **Physics-aware coordinate baseline:** shock position, separation point, recirculation length, thermal boundary-layer thickness 같은 feature로 field를 align한 뒤 FNO/GNO 성능 변화를 본다.
