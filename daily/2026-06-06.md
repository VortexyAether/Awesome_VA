# Daily Research Radar — 2026-06-06

## Highlights

- **오늘의 핵심 신호는 “surrogate의 정확도보다 deployment contract가 더 중요해진다”입니다.** PINO training, WSS inverse reconstruction, underwater CFD path-planning surrogate, thermal-hydraulic multi-agent governance가 모두 field prediction을 넘어서 관측·검증·운영 전환을 다룹니다.
- **Neural operator는 구조적 inductive bias 경쟁으로 들어갔습니다.** LiNO는 reflection/refraction/scattering analogy, EqGINO는 3D geometry-aware equivariance, FreqNO-DPS는 sparse sensor + diffusion posterior correction으로 FNO류의 약점을 직접 겨냥합니다.
- **CAD-AI는 representation layer가 빠르게 두꺼워지는 중입니다.** Attributed Feature Graphs, UniCAD, BRepCLIP은 “3D shape”가 아니라 CAD feature/B-Rep/멀티모달 task를 모델 입력으로 삼아 CAD-to-CAE 자동화와 더 가까운 표현을 만듭니다.
- **MCP/agent protocol 쪽은 실험실 인프라 관점에서 볼 만합니다.** LAP는 agent-to-instrument edge를, MCP server fault/DCI 논문은 tool description과 실제 구현의 불일치 위험을 보여줍니다. engineering agent는 capability보다 auditability가 먼저입니다.
- **저신호 generic agent 논문은 제외했습니다.** sciML/CFD/CAD/control/engineering automation과 직접 연결되는 항목만 남겼습니다.

## Added Resources

### SciML / CFD / thermal-fluid surrogates

- [Wall Shear Stress Reconstruction from Concentration: Differentiable Physics and Physics-Informed Neural Networks](https://arxiv.org/abs/2606.06313) — passive scalar concentration/temperature 관측에서 WSS를 복원하는 inverse-flow 문제를 differentiable physics adjoint와 PINN으로 비교합니다. CFD post-processing/실험 계측에서 “속도장 전체 없이 hidden flow metric을 복원”하는 패턴이라 thermal/fluid inverse workflow에 유용합니다. `#fluid-dynamics` `#inverse-problem` `#PINN` `#differentiable-physics`
- [On the training of physics-informed neural operators for solving parametric partial differential equations](https://arxiv.org/abs/2606.06164) — PINO를 paired simulation data 없이 physics residual로 안정적으로 학습시키는 문제를 training protocol 관점에서 정리합니다. neural operator를 실제 PDE family에 붙일 때 loss/optimization/robustness가 병목이라는 점을 다시 확인시켜 줍니다. `#PINO` `#neural-operator` `#PDE` `#training`
- [3D Underwater Path Planning via Generative Flow Field Surrogates](https://arxiv.org/abs/2606.06077) — RANS CFD로 얻은 3D propeller wake를 cGAN 기반 flow-field surrogate로 만들어 AUV launch/recovery path planning에 씁니다. “CFD surrogate → control/planning decision”까지 닿는 적용 사례라 단순 field emulator보다 신호가 좋습니다. `#CFD-surrogate` `#RANS` `#path-planning` `#control`
- [Validation-Gated Multi-Agent Governance for Online Adaptation of Thermal-Hydraulic Surrogate Models under Operating-Regime Shift](https://arxiv.org/abs/2606.03321) — thermal-hydraulic surrogate의 online regime shift 대응을 Monitor/Diagnosis/Adaptation/Safety-Auditor/Orchestrator agent와 deterministic champion-challenger gate로 관리합니다. 실전 digital twin에서는 agent가 최종 결정자가 아니라 auditable reviewer여야 한다는 좋은 패턴입니다. `#thermal-hydraulic` `#surrogate` `#multi-agent` `#validation-gate`

### Neural operators / operator diagnostics

- [EqGINO: Equivariant Geometry-Informed Fourier Neural Operators for 3D PDEs](https://arxiv.org/abs/2606.03260) — 3D PDE surrogate가 geometry transformation에 취약한 문제를 geometry-informed FNO와 equivariance로 다룹니다. CAD/mesh-to-field surrogate에서 좌표계·회전·geometry shift robustness를 평가할 때 참고할 만합니다. `#FNO` `#equivariance` `#3D-PDE` `#geometry`
- [Let There Be Light: Reflection, Refraction and Scattering for Neural Operators](https://arxiv.org/abs/2606.03262) — LiNO는 light propagation analogy로 nonlocal communication, mesh scalability, interpretability, compute cost 사이의 trade-off를 줄이려는 neural operator입니다. 아직 검증은 봐야 하지만 operator architecture search 신호로는 좋습니다. `#neural-operator` `#LiNO` `#PDE` `#architecture`
- [Correcting Neural Operator Spectral Bias via Diffusion Posterior Sampling with Sparse Observations](https://arxiv.org/abs/2606.03936) · [code](https://github.com/niccoloperrone/FreqNO-DPS) — neural operator가 high-frequency를 죽이는 spectral bias를 sparse observations + diffusion posterior sampling으로 보정합니다. CFD/thermal surrogate에서 field RMSE만 보고 fine-scale loss를 놓치지 않기 위한 correction/evaluation reference입니다. `#spectral-bias` `#diffusion` `#sparse-sensors` `#neural-operator`

### CAD / engineering automation / agent protocols

- [Bridging CAD and Data-Driven Design: Attributed Feature Graphs for Engineering Design](https://arxiv.org/abs/2606.06405) — CAD feature를 extrusion/rib/pocket 같은 node와 relation으로 표현하는 Attributed Feature Graphs를 제안합니다. CAD-to-CAE surrogate가 geometry를 point cloud로만 보지 않고 design intent/feature semantics를 유지하는 방향이라 중요합니다. `#CAD` `#feature-graph` `#design-automation` `#CAE`
- [UniCAD: A Unified Benchmark and Universal Model for Multi-Modal Multi-Task CAD](https://arxiv.org/abs/2606.05058) — point-to-CAD reconstruction, text/image-to-CAD generation, CAD QA를 한 benchmark/model로 묶습니다. CAD foundation model 쪽이 task별 demo에서 평가 체계로 넘어가는 신호입니다. `#CAD` `#benchmark` `#multimodal` `#text-to-CAD`
- [BRepCLIP: Contrastive Multimodal Pretraining on BRep Primitives for CAD Understanding](https://arxiv.org/abs/2606.05515) — native CAD B-Rep의 face/edge primitive를 language/image embedding과 정렬합니다. downstream CAE에는 mesh보다 B-Rep/feature-level representation이 더 유용한 경우가 많아서 추적 가치가 있습니다. `#BRep` `#CAD-understanding` `#contrastive-learning` `#geometry`
- [LAP: An Agent-to-Instrument Protocol for Autonomous Science](https://arxiv.org/abs/2606.03755) — MCP가 agent-to-tool, A2A가 agent-to-agent edge를 다룬다면 LAP는 lab instrument와 agent 사이의 protocol layer를 겨냥합니다. 실험·계측·simulation 장비까지 포함하는 autonomous engineering workflow의 interface 후보로 볼 만합니다. `#agent-protocol` `#autonomous-science` `#instrumentation` `#MCP-adjacent`
- [Description-Code Inconsistency in Real-world MCP Servers](https://arxiv.org/abs/2606.04769) — MCP server의 자연어 tool description과 실제 code behavior가 어긋나는 DCI 문제를 측정/탐지합니다. CAD/CAE agent가 tool description만 믿고 위험한 작업을 하는 것을 막으려면 spec-code consistency와 runtime audit이 필요합니다. `#MCP` `#tool-safety` `#agent-reliability` `#audit`
