# Daily Research Radar — 2026-06-04

## Highlights

- **오늘의 핵심 신호는 “surrogate deployment에는 validation gate가 필요하다”입니다.** Thermal-hydraulic surrogate adaptation, FreqNO-DPS, PINN-residual AMR은 모두 모델을 단순 예측기가 아니라 관측·검증·교정 loop 안의 부품으로 다룹니다.
- **Sim-to-real fluid foundation model이 실험 데이터 쪽으로 이동합니다.** Rayleigh-Taylor turbulence에서 simulation-only fine-tuning 후 laboratory growth band로 넘어가는 사례는 foundation model 주장 중 꽤 강한 신호입니다.
- **CAD/CAE agent는 생성보다 검증-first입니다.** VFEAgent와 MUSE는 각각 FEA 자동화와 text-to-CAD benchmark에서 executable code, valid geometry, physical validity, manufacturability/assemblability를 평가축으로 세웁니다.
- **Classical solver를 버리지 않는 SciML 패턴이 좋습니다.** PINN residual을 AMR indicator로 쓰거나 DiffFEM을 PINN 대안으로 삼는 논문들은 “neural = solver replacement”보다 더 실무적인 방향을 보여줍니다.
- **Engineering workflow automation은 structured context가 먼저입니다.** Knowledge graph 기반 virtual commissioning agent와 AAS→PDDL 변환은 raw prompt가 아니라 PLC/CAD/digital-twin metadata를 구조화한 뒤 agent/planner를 붙이는 패턴입니다.

## Added Resources

### Thermal / neural operators / surrogate governance

- [Validation-Gated Multi-Agent Governance for Online Adaptation of Thermal-Hydraulic Surrogate Models under Operating-Regime Shift](https://arxiv.org/abs/2606.03321) — thermal-hydraulic surrogate를 deployment 이후에도 Monitor/Diagnosis/Adaptation/Safety-Auditor/Orchestrator agent와 deterministic champion-challenger gate로 관리합니다. 실전 digital twin에서는 agent가 결정권자가 아니라 auditable reviewer이고, 최종 교체 권한은 검증 gate가 가진다는 점이 핵심입니다. `#thermal-hydraulic` `#surrogate` `#multi-agent` `#validation-gate`
- [Correcting Neural Operator Spectral Bias via Diffusion Posterior Sampling with Sparse Observations](https://arxiv.org/abs/2606.03936) · [code](https://github.com/niccoloperrone/FreqNO-DPS) — neural operator가 high-frequency를 죽이는 spectral bias를 sparse sensor + diffusion posterior sampling + frequency-shaped guidance로 보정합니다. CFD/thermal surrogate 평가에서 field RMSE만 보면 fine-scale loss를 놓치므로, spectral calibration 관점이 중요합니다. `#neural-operator` `#spectral-bias` `#diffusion` `#sparse-sensors`

### Turbulence / experimental data assimilation

- [Emergent Transfer of a Physics Foundation Model from Simulation to Laboratory Turbulence](https://arxiv.org/abs/2606.01470) — continuum-dynamics foundation model Walrus를 소수 DNS RTI case로 fine-tune한 뒤 laboratory turbulence data에 zero-shot 적용합니다. simulation-trained model이 noisy experiment regime으로 넘어가 오래된 sim-experiment gap을 설명하는 데 쓰인다는 점이 고신호입니다. `#turbulence` `#foundation-model` `#sim-to-real` `#RTI`
- [Uncovering Turbulent Dynamics in Stenotic Flows from 4D-flow MRI Measurements via Resolvent Analysis and Data Assimilation](https://arxiv.org/abs/2606.03838) — 4D-flow MRI 측정값을 PINN 기반 data assimilation으로 보정하고 mean pressure/eddy viscosity를 추정한 뒤 LSA/resolvent analysis를 수행합니다. sparse/noisy experimental flow data를 modal analysis 가능한 physics-consistent field로 바꾸는 workflow 참고축입니다. `#data-assimilation` `#resolvent-analysis` `#MRI` `#turbulence`

### Differentiable solvers / AMR / inverse analysis

- [Physics-Informed Residuals for Adaptive Mesh Refinement in Finite-Difference PDE Solvers](https://arxiv.org/abs/2606.02475) — PINN을 final solver로 쓰지 않고 off-grid residual probe로 써서 finite-difference AMR을 유도합니다. classical solver를 유지하면서 neural diagnostic으로 mesh placement를 개선하는 현실적 SciML 패턴입니다. `#AMR` `#PINN` `#finite-difference` `#solver-assist`
- [Critical evaluation of PINN for FWD inverse analysis and differentiable FEM as an alternative](https://arxiv.org/abs/2606.03210) — layered pavement inverse analysis에서 PINN/XPINN이 loss weighting, architecture, noise에 민감하고 DiffFEM이 더 안정적이라는 결과입니다. differentiable forward solver가 있으면 soft-constraint PINN보다 hard-constraint DiffFEM이 낫다는 좋은 경고입니다. `#DiffFEM` `#PINN` `#inverse-analysis` `#engineering-mechanics`

### CAD / CAE / intelligent engineering workflows

- [VFEAgent: A Multimodal Agent Framework for End-to-End Automated Finite Element Analysis](https://arxiv.org/abs/2605.28978) — image/problem description에서 FEA specification을 뽑고 verification-first code synthesis로 실행 가능하고 물리적으로 유효한 simulation을 만들려는 multi-agent framework입니다. CFD/CAE agent도 “생성”보다 structured spec, self-debugging, fallback, validation이 핵심이라는 신호입니다. `#FEA` `#multimodal-agent` `#verification` `#CAE-automation`
- [MUSE: Benchmarking Manufacturable, Functional, and Assemblable Text-to-CAD Generation](https://arxiv.org/abs/2605.28579) · [project](https://dong7313.github.io/muse-benchmark/) — text-to-CAD를 code check → geometry check → design-intent alignment로 평가하고 manufacturability/functionality/assemblability까지 봅니다. CAD-to-CAE 자동화에서는 그럴듯한 shape보다 executable B-Rep assembly와 engineering intent가 중요합니다. `#CAD` `#benchmark` `#manufacturability` `#assemblability`
- [Multi-Agent Framework Leveraging Knowledge Graphs for Virtual Commissioning Models](https://arxiv.org/abs/2606.03255) — TIA Portal/NX MCD 정보를 graph database로 구조화한 뒤 system understanding, simulation component generation, PLC-simulation mapping을 agent가 지원합니다. engineering agent에는 shared structured context가 먼저라는 좋은 사례입니다. `#knowledge-graph` `#multi-agent` `#virtual-commissioning` `#industrial-automation`
- [From Capability Models to Automated Planning: An AAS-Native Approach for Automatic PDDL Generation](https://arxiv.org/abs/2606.02167) — Asset Administration Shell capability model에서 PDDL planning problem을 자동 생성합니다. CAD/CAE workflow도 structured asset/case metadata에서 planner-ready representation을 생성하는 방향으로 생각할 수 있습니다. `#digital-twin` `#PDDL` `#AAS` `#planning`

## Best Find of the Day

**Validation-Gated Multi-Agent Governance for Online Adaptation of Thermal-Hydraulic Surrogate Models**를 오늘의 best find로 봅니다. 이유는 architecture보다 운영 원칙이 좋기 때문입니다. Surrogate가 operating envelope 밖으로 나갈 때 agent가 곧장 모델을 바꾸는 게 아니라, 역할 분리된 agent council이 진단·제안·감사를 하고 deterministic champion-challenger gate가 최종 배포 권한을 갖습니다. CFD/thermal digital twin을 실제 제어·감시 loop에 넣으려면 이 “agent는 보조, gate가 권한” 구조가 훨씬 안전합니다.

## Possible Research Ideas

1. **Thermal/CFD surrogate validation contract:** 모델 artifact에 operating envelope, spectral error, sensor reconstruction, OOD trigger, champion-challenger score, rollback rule을 포함하는 배포 템플릿을 만든다.
2. **Spectral-bias dashboard for flow surrogates:** FreqNO-DPS를 참고해 FNO/DeepONet/UNet surrogate의 wavenumber-band별 error와 sensor-conditioned correction 가능성을 비교한다.
3. **PINN-as-diagnostic AMR experiment:** OpenFOAM/Python PDE case에서 PINN residual 또는 cheap neural residual probe가 refinement/sampling/re-run 위치를 얼마나 잘 고르는지 gradient indicator와 비교한다.
4. **CAD-to-CAE validation cascade:** MUSE/VFEAgent식으로 CAD agent output을 code validity → geometry validity → meshability → boundary-condition completeness → solver run → physical sanity check 순서로 평가한다.
5. **Engineering metadata graph:** FreeCAD/OpenFOAM case folder의 geometry, mesh, BC, solver logs, post-processing plots를 graph/JSON으로 정리한 뒤 agent가 수정 제안을 grounded하게 하도록 만든다.
