# Daily Research Radar — 2026-05-31

## Highlights

- **주말이라 신규 arXiv sciML 물량은 얇지만, 이번 묶음의 핵심은 “agent가 연구/설계를 판단할 때 무엇을 gate로 삼을 것인가”입니다.** SoundnessBench와 SchGen은 각각 연구 아이디어 평가와 하드웨어 설계 생성에서 representation·검증 기준이 모델 크기보다 중요하다는 신호를 줍니다.
- **CFD/thermal 쪽은 surrogate의 실사용 조건이 더 선명해졌습니다.** SMR thermal-hydraulic digital twin, CFDTwin, automotive transfer learning, HiLiftAeroML은 모두 “모델 정확도”보다 geometry/domain shift, GUI/tooling, dataset contract, operating envelope를 묻습니다.
- **CAD/engineering agent는 demo generation에서 validated workflow로 이동 중입니다.** Physics-in-the-Loop, Self-improving CAD agents, Hylos 계열은 CAD artifact가 실제 CAE/robotics substrate가 되려면 FEA/physics checks, entity/frame/action contract, rollback/review boundary가 필요하다는 같은 결론으로 수렴합니다.
- **LinkedIn 주제 후보는 꽤 좋습니다.** “AI가 연구 아이디어를 너무 낙관적으로 평가한다”와 “CAD/CFD agent의 핵심은 생성 능력이 아니라 validation contract”를 한 글로 엮을 수 있습니다.

## Added Resources

### Research-agent evaluation & design automation

- [SoundnessBench: Can Your AI Scientist Really Tell Good Research Ideas from Bad Ones?](https://arxiv.org/abs/2605.30329) — ICLR submission에서 복원한 1,099개 ML 연구 제안의 soundness benchmark입니다. 12개 frontier LLM이 낮은 soundness 제안을 과하게 긍정하는 optimism bias를 보였다는 점이 핵심입니다. VA식 sciML 연구 agent에도 “아이디어 생성”보다 proposal-stage 방법론 검증 gate가 필요하다는 경고입니다. `#AI-scientist` `#research-evaluation` `#soundness` `#agent-guardrail`
- [SchGen: PCB Schematic Generation with Semantic-Grounded Code Representations](https://arxiv.org/abs/2605.30345) — natural-language request에서 editable PCB schematic을 생성하기 위해 tool-specific geometry syntax 대신 semantic editing primitive와 pin-name wiring 중심 표현을 제안합니다. 직접 CFD/CAD는 아니지만, 복잡한 engineering artifact는 representation을 잘 잡아야 LLM이 기능적 correctness를 맞춘다는 점에서 CAD/CAE automation에 참고할 만합니다. `#design-automation` `#hardware` `#representation` `#LLM`

### CFD / thermal-fluid surrogate workflow

- [Neural Operator-Based Surrogate Model for CFD: Helical Coil Steam Generator in Small Modular Reactor](https://arxiv.org/abs/2605.30277) — SMR helical coil steam generator transient CFD를 digital twin에 넣기 위해 AE/CAE ROM과 L-DeepONet/FNO를 비교합니다. 열수력 장치처럼 복잡 형상·transient field·operating envelope가 있는 문제에서 surrogate asset을 어떻게 구성할지 좋은 기준점입니다. `#CFD` `#thermal-hydraulics` `#SMR` `#neural-operator`
- [CFDTwin: An open-source GUI and Python toolkit for POD-NN surrogate modeling of ANSYS Fluent simulations](https://arxiv.org/abs/2605.27725) — Fluent 결과를 POD-NN surrogate로 바꾸는 GUI/Python toolkit입니다. 최첨단 architecture보다 기존 CFD 사용자가 surrogate를 만들고 반복 설계/디지털트윈 workflow에 넣는 경로를 제공한다는 점이 실무적으로 큽니다. `#CFD` `#Fluent` `#POD-NN` `#digital-twin`
- [Adapting Automotive Aerodynamics Surrogates to New Vehicle Families via Transfer Learning](https://arxiv.org/abs/2605.27968) — automotive CFD surrogate가 다른 vehicle family로 넘어갈 때 geometry encoder representation이 전이되는지 leave-one-family-out으로 검증합니다. CAD/CFD design loop에서 제일 위험한 지점은 random split accuracy가 아니라 family-level OOD shift입니다. `#aerodynamics` `#transfer-learning` `#OOD` `#CFD-surrogate`
- [HiLiftAeroML: High-Fidelity Computational Fluid Dynamics Dataset for High-Lift Aircraft Aerodynamics](https://arxiv.org/abs/2605.19565) — NASA CRM high-lift geometry variants와 angles of attack 조합의 공개 high-fidelity CFD dataset입니다. 장난감 airfoil을 넘어 geometry-variant surrogate benchmark를 만들 수 있다는 점에서 계속 추적할 가치가 큽니다. `#CFD-dataset` `#aerodynamics` `#benchmark` `#surrogate`

### Agentic CAD / validated engineering workflow

- [Physics-in-the-Loop: A Hybrid Agentic Architecture for Validated CAD Engineering Design](https://arxiv.org/abs/2605.19717) — LLM CAD generation에 validated knowledge-based engineering tools를 decision loop 안에 넣자는 architecture입니다. “LLM이 물리를 배웠을 것”이라고 믿지 말고, CAD→mesh→thermal/CFD pipeline에 artifact gate를 넣어야 한다는 메시지가 분명합니다. `#CAD-agent` `#physics-validation` `#CAE` `#agentic-engineering`
- [Self-Improving CAD Generation Agents with Finite Element Analysis as Feedback](https://arxiv.org/abs/2605.17448) — multi-part STEP 생성 후 FEA feedback으로 repair loop를 돌리는 CAD agent 작업입니다. 첫 시도 strict pass가 사실상 0에 가깝다는 결과가 오히려 중요합니다: engineering agent는 예쁜 shape보다 simulation feedback과 수정 루프가 핵심입니다. `#CAD` `#FEA` `#STEP` `#validation`

## Best Find of the Day

**SoundnessBench**를 오늘의 best find로 봅니다. CFD/CAD agent도 결국 “좋은 설계안/연구 아이디어를 고르는 문제”를 피할 수 없는데, LLM이 proposal soundness를 과하게 긍정한다면 자동화 pipeline은 쓸데없는 CFD 계산과 잘못된 설계 탐색에 시간을 태우게 됩니다. 생성 agent보다 먼저 필요한 것은 냉정한 first-gate evaluator입니다.

## Possible Research Ideas

1. **SciML proposal soundness gate:** CFD surrogate 아이디어를 입력하면 novelty, data availability, validation metric, OOD split, compute budget, physical failure mode를 체크하는 rubric을 만든다.
2. **Representation-first CAD/CAE agent:** SchGen의 semantic primitive 관점을 CadQuery/FreeCAD operation representation에 적용해, geometry-heavy syntax보다 constraint/action 중심 중간표현을 설계한다.
3. **Surrogate asset readiness score:** CFDTwin/SMR/automotive transfer 사례를 묶어 geometry coverage, operating envelope, validation plots, UI/API, uncertainty, OOD test를 점수화한다.
4. **FEA/CFD repair loop benchmark:** CAD agent 출력물을 STEP validity → meshability → simple FEA/thermal solve → repair prompt 순서로 돌리는 작은 benchmark를 만든다.
5. **LinkedIn synthesis:** “AI engineering agents fail less from lack of generation and more from weak validation contracts”라는 주제로 SoundnessBench + CAD/CFD 사례를 엮는다.

## Slack-ready copy
