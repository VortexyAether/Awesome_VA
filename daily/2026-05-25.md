# Daily Research Radar — 2026-05-25

## Highlights

- **오늘의 핵심 신호는 “surrogate가 좋아졌다”보다 데이터셋·검증·편집 가능한 geometry가 같이 올라온다는 점입니다.** HiLiftAeroML은 high-lift aircraft의 공개 high-fidelity CFD dataset을 내고, SDFStent는 patient-specific CFD 전 단계인 geometry deformation을 실시간화합니다.
- **CFD surrogate는 conditioning을 더 유연하게 만드는 방향이 강합니다.** Inpainting Physics는 명시 boundary/geometry input에 묶인 forward surrogate 대신 context-driven completion을 제안하고, ViT-K는 coupled free-flow/porous-media interface problem을 few-shot으로 다룹니다.
- **CAD agent 쪽은 memory·validation·closed-loop search가 계속 중요합니다.** Memory-augmented RL CAD generation은 긴 CAD operation sequence의 상태 기억 문제를 정면으로 다루고, 기존 Physics-in-the-Loop/FEA-feedback 흐름과 함께 “생성 모델 단독”보다 tool/solver feedback loop가 핵심임을 보여줍니다.
- **설계 자동화의 좋은 패턴은 field prediction이 아니라 engineering action으로 끝납니다.** SDFStent와 robotic disassemblability fastener reduction 계열은 해석/계획 결과를 geometry edit 또는 설계 단순화 제안으로 되돌립니다.
- **오픈소스 도구는 아직 high-signal만 선별해야 합니다.** PCB design agent류는 CAD/EDA automation 아이디어는 좋지만, VA의 CFD/thermal/CAD pipeline에는 “검증 루프 구조 참고” 수준으로 보는 게 안전합니다.

## Added Resources

### CFD datasets, surrogates & geometry-conditioned simulation

- [HiLiftAeroML: High-Fidelity Computational Fluid Dynamics Dataset for High-Lift Aircraft Aerodynamics](https://arxiv.org/abs/2605.19565) — NASA CRM high-lift geometry variants와 angles of attack 조합으로 만든 공개 high-fidelity CFD dataset. aerodynamic surrogate가 장난감 geometry를 벗어나려면 이런 geometry-variant dataset이 필요합니다. `#CFD-dataset` `#aerodynamics` `#surrogate` `#high-lift`
- [SDFStent: Real-time interactive virtual stenting via SDF deformation fields](https://arxiv.org/abs/2605.22009) — stent-induced vessel geometry deformation을 SDF field로 실시간 예측해 patient-specific CFD geometry preparation을 줄이려는 작업. “CFD surrogate”보다 앞단의 geometry update를 interactive하게 만드는 점이 고신호입니다. `#CFD-preprocessing` `#SDF` `#biomedical-CFD` `#geometry`
- [Inpainting physics: self-supervised learning for context-driven fluid simulation](https://arxiv.org/abs/2605.08832) — boundary condition/geometry specification에 고정된 forward CFD surrogate 대신, 주변 context로 빠진 flow region을 self-supervised inpainting하는 접근. BC shift와 local geometry change에 강한 surrogate 설계 아이디어로 볼 만합니다. `#self-supervised` `#fluid-simulation` `#inpainting` `#surrogate`
- [ViT-K: A Few-Shot Learning Model for Coupled Fluid-Porous Media Flows with Interface Conditions](https://arxiv.org/abs/2605.13912) — Stokes/Navier–Stokes–Darcy coupled flow의 interface condition을 few-shot으로 다루는 모델. porous/filtration/physiological transport처럼 경계면 physics가 중요한 문제에서 유용한 reference입니다. `#few-shot` `#porous-media` `#coupled-flow` `#interface-condition`
- [Compositional Neural Operators for Multi-Dimensional Fluid Dynamics](https://arxiv.org/abs/2605.11691) — universal SFM 하나보다 physics/operator block을 조합하는 방식으로 multi-dimensional fluid dynamics generalization을 노리는 논문. multi-regime CFD foundation model의 negative transfer 문제를 피하는 설계 방향으로 중요합니다. `#neural-operator` `#compositionality` `#fluid-dynamics` `#foundation-model`
- [Reduced-Order Modeling of Parameterized Visco-Plastic Shallow Flows](https://arxiv.org/abs/2605.06526) — Herschel–Bulkley visco-plastic shallow-water flows에 대한 non-intrusive ROM. moving fronts/yield surfaces처럼 비매끄러운 rheology가 있는 surrogate benchmark로 쓸 만합니다. `#ROM` `#visco-plastic` `#shallow-flow` `#surrogate`

### CAD, design automation & agentic engineering tools

- [Memory-Augmented Reinforcement Learning Agent for CAD Generation](https://arxiv.org/abs/2605.19748) — 복잡한 CAD model의 긴 operation sequence와 geometric constraints를 memory-augmented RL로 다루려는 논문. CAD agent가 “현재까지 만든 feature history를 안정적으로 기억/수정”해야 한다는 문제를 잘 드러냅니다. `#CAD-agent` `#reinforcement-learning` `#memory` `#design-automation`
- [assalas/pcb-designer-ai-agent](https://github.com/assalas/pcb-designer-ai-agent) — KiCad/Altium/Eagle 연동을 표방하는 AI PCB design automation repo. CFD/CAD 직접 도구는 아니지만, placement/routing/signal-integrity check를 agent loop에 넣는 EDA automation 구조는 engineering validation loop 참고용입니다. `#engineering-agent` `#EDA` `#design-automation` `#open-source`

## Best Find of the Day

**HiLiftAeroML**입니다. 이유는 간단합니다: CFD-AI에서 모델 논문보다 더 부족한 것이 “실제 engineering geometry variation을 가진 공개 high-fidelity dataset”입니다. NASA CRM high-lift 계열은 geometry, AoA, mesh/solver 신뢰성, surrogate benchmark를 한 번에 엮을 수 있어 VA의 CAD→CFD→surrogate 관심사와 잘 맞습니다.

## Possible Research Ideas

1. **Geometry-variant aerodynamic surrogate benchmark:** HiLiftAeroML을 기준으로 FNO/Graph surrogate/latent diffusion을 비교하되, random split이 아니라 geometry-family holdout과 AoA extrapolation을 따로 본다.
2. **Context-conditioned flow completion toy project:** cavity/duct/porous interface 데이터에서 일부 field와 boundary metadata만 주고 missing region을 reconstruct하는 “inpainting physics” baseline을 만든다.
3. **CAD operation memory stress test:** CadQuery/FreeCAD script를 길게 생성·수정하는 agent에게 feature count, dependency break, export validity, meshability를 score로 주고 memory-augmented prompting/agent-state가 실제로 도움이 되는지 본다.
