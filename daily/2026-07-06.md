## New Papers / Tools

### SNAP-FM

- Link: https://arxiv.org/abs/2607.00095
- Type: Paper / constrained generative modeling for physical simulation
- Why it matters: Generative surrogates can violate conservation laws, boundary conditions, or nonlinear invariants. SNAP-FM treats constrained sampling as a sparse nonlinear optimization problem and exploits block-sparse Jacobian/KKT structure with ExaModels.jl, MadNLP.jl, and GPU sparse factorization.
- VA relevance: Good citation for “AI solver output is a draft until constraint projection / residual gates / fallback contracts verify it.”
- Caveat: Method paper; industrial CFD geometry, noisy observations, and public reproduction surface still need checking.
- Action: Cite

### ContextNest / ContextNext

- Link: https://arxiv.org/abs/2607.02116
- Type: Paper / open specification for context governance
- Why it matters: Engineering agents need to prove what they read, which version it was, and whether it can be reconstructed later. ContextNest puts approved/current/attributable/hash-verified context selection beneath RAG and MCP-style live sources.
- VA relevance: Strong vocabulary for Obsidian → repo → solver/CAD agent workflows where provenance and audit trails matter as much as retrieval relevance.
- Caveat: Title says ContextNest while abstract also says ContextNext; implementation maturity and repo release need separate verification.
- Action: Read/Save

### Self-explainable Operator Learning

- Link: https://arxiv.org/abs/2607.02203
- Type: Paper / interpretable operator learning
- Why it matters: The method reformulates operator learning through additive integral-equation components, so localized input subdomains can be connected to output patterns. This is closer to engineering explanation than a heatmap added after the fact.
- VA relevance: Useful for CFD/biomedical/aero surrogates where sparse sensing and “which spatial feature drove the prediction?” are trust questions.
- Caveat: Blood-flow and unsteady-aero examples are promising but not industrial turbulent CFD validation.
- Action: Cite
