## Added Resources

### CFD / neural operators / digital twins / control

1. [A Validated LBM Dataset and Pipeline for Surrogate Modeling of Turbulent 3D Obstructed Channel Flows](https://arxiv.org/abs/2606.16765) — generated 3D obstructed-channel geometries, Re=1,000–10,000, cumulant LBM, grid convergence, experimental checks(Strouhal/drag/turbulent fluctuations)를 포함한 surrogate training-data pipeline입니다. **VA에 중요한 이유:** neural operator 성능 비교가 “예쁜 flow field”가 아니라 검증된 turbulent dataset + physics-informed metric + energy cascade 보존으로 가야 한다는 기준점입니다. **Action: Test** `#CFD` `#turbulence` `#LBM` `#surrogate-dataset`
2. [PhysGuard: Fisher-Guided Gradient Projection for Sim-to-Real Neural PDE Surrogates](https://arxiv.org/abs/2606.16602) — simulation-pretrained neural operator를 소량 experimental/real data에 맞출 때 physics-critical parameter direction을 Fisher 정보로 보호하는 adaptation 방법입니다. **VA에 중요한 이유:** CFD/thermal surrogate를 lab data로 fine-tune할 때 pretraining에서 배운 물리 구조를 망가뜨리지 않는 “safe adaptation” layer가 필요합니다. **Action: Test** `#sim-to-real` `#neural-operator` `#adaptation` `#validation`
3. [Learning Interface Breakup: A Geometry-Conditioned Latent Surrogate for Spray Formation](https://arxiv.org/abs/2606.16587) — nozzle geometry가 transient two-phase breakup을 어떻게 바꾸는지, full VOF state 대신 AMR cell-density field를 latent proxy로 써서 surrogate를 만드는 접근입니다. **VA에 중요한 이유:** geometry-conditioned multiphase surrogate는 CAD→CFD design loop와 직접 닿고, adaptive mesh/resolution pattern 자체를 학습 신호로 쓰는 점이 흥미롭습니다. **Action: Read** `#two-phase-flow` `#spray` `#geometry-conditioned` `#AMR`
4. [Graphical conditional generative modeling for digital twin modeling](https://arxiv.org/abs/2606.16219) — digital twin의 open-ended fidelity 문제를 줄이기 위해 QoI에 필요한 변수만 조건부 분포 관점으로 찾는 stochastic surrogate framework입니다. **VA에 중요한 이유:** 평균 예측만이 아니라 variability/tail/multimodality까지 조건부 dependency를 보자는 점이 uncertainty-aware twin 설계와 잘 맞습니다. **Action: Save** `#digital-twin` `#uncertainty` `#generative-modeling` `#QoI`
5. [Decision-Weighted Flow Matching for Contextual Stochastic Optimization](https://arxiv.org/abs/2606.16790) — scenario generator의 distributional fit 대신 downstream regret/decision sensitivity로 flow-matching objective를 reweight합니다. **VA에 중요한 이유:** surrogate나 generative model은 field/sample fidelity보다 최종 설계·운영 decision을 얼마나 망치지 않는지가 중요하므로, decision-weighted training lens가 유용합니다. **Action: Watch** `#flow-matching` `#stochastic-optimization` `#decision-quality` `#surrogate`
6. [Learning practically stabilizing output-feedback nonlinear controllers](https://arxiv.org/abs/2606.16930) — 비싼 nonlinear controller-observer pair를 recurrent surrogate controller로 모사하면서 Lyapunov candidate를 함께 학습해 practical stability를 유도합니다. **VA에 중요한 이유:** CFD/thermal surrogate를 control loop에 넣을 때 imitation accuracy만으로는 부족하고, stability/constraint satisfaction validation이 필요합니다. **Action: Read** `#control` `#surrogate-controller` `#Lyapunov` `#stability`

### CAD / engineering agents / workflow evaluation

7. [LabOSBench: Benchmarking Computer Use Agents for Scientific Instrument Control](https://arxiv.org/abs/2606.16802) — 실제 고가 장비 대신 web-based scientific-instrument simulator에서 multimodal GUI agents를 평가하는 benchmark입니다. **VA에 중요한 이유:** CAE/CAD/실험 agent도 “버튼을 눌렀다”가 아니라 feedback-driven parameter adjustment와 safety/reproducibility를 평가해야 합니다. **Action: Watch** `#scientific-agents` `#benchmark` `#instrument-control` `#GUI-agent`

[LLM-based Visual Code Completion for Aerospace Geometric Design](https://arxiv.org/abs/2606.16806) — aerospace visual programming copilot, Wingbuilder Grasshopper plugin, Aerospace Visual Programming Dataset(AVPD)을 제안합니다. **VA에 중요한 이유:** aerospace CAD automation은 text-to-shape보다 explainable visual program + domain-specific geometry abstraction + task dataset 쪽이 더 실용적입니다. **Action: Save** `#CAD` `#aerospace` `#visual-programming` `#geometry-automation`
9. [AI+CAD Data Representation Architecture](https://arxiv.org/abs/2606.16797) — AI+CAD에서 “visual plausibility”보다 industrial-grade parametric feature modeling과 data representation architecture가 더 근본적이라고 주장하는 survey/position 성격의 논문입니다. **VA에 중요한 이유:** CAD/CAE 자동화는 mesh preview가 아니라 editable feature history, constraints, BRep/parametric semantics, downstream manufacturability/meshing까지 보존해야 합니다. **Action: Cite** `#CAD` `#BRep` `#parametric-modeling` `#representation`

[The CREATOR Project: Towards a Computational Electric Machine Laboratory](https://arxiv.org/abs/2606.16653) — electric machine design을 위해 EM, fluid dynamics, materials, geometry, optimization을 통합하는 large-scale computational laboratory vision입니다. **VA에 중요한 이유:** “single surrogate paper”보다 multiphysics design lab의 모델·데이터·optimization stack를 어떻게 묶는지 보는 architecture reference입니다. **Action: Watch** `#multiphysics` `#design-automation` `#electric-machines` `#computational-lab`

## Suggested Experiments / Follow-up

1. **Turbulent surrogate dataset checklist:** geometry generation, Reynolds coverage, solver verification, grid convergence, experimental comparison, energy spectrum/cascade metric, rollout horizon을 manifest로 만든다.
2. **Sim-to-real adaptation smoke test:** FNO/UNO 계열에서 naive fine-tune vs PhysGuard-style protected update를 heat equation 또는 simple flow lab/noisy-observation setting으로 비교한다.
3. **Decision-quality metric:** surrogate field error와 downstream design/control regret을 따로 기록해서 “RMSE는 낮지만 decision은 틀리는” case를 찾는다.
4. **CAD representation rubric:** text/image-to-CAD 항목을 mesh-only, BRep, feature tree, parametric program, constraint-preserving edit 가능성으로 등급화한다.
