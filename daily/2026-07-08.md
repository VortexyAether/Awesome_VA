## New Papers / Tools

### GaP — Graph-as-Policy multi-agent self-learning harness

- Link: https://arxiv.org/abs/2607.05369
- Project: https://graph-robots.github.io/gap
- Code: https://github.com/graph-robots/graph-as-policy
- Type: Paper / robotics-agent harness / simulation-rehearsed workflow graph
- Why it matters: Variational Automation task를 perception/planning/control node로 구성된 directed computation graph로 만들고, 내부 simulation rehearsal에서 실패를 localize·수정합니다. CAE/CFD로 옮기면 `solver workflow graph 생성 → case-family rehearsal → residual/constraint/success metric으로 graph 갱신`이라는 설계 언어가 됩니다.
- Evidence: project page reports 8 open VA benchmarks, graph execution in sim/real, Make Popcorn self-learning from 33% initial success to 94% simulation and 90% real robot, and caveats around quasi-static tasks and industrial cycle time.
- Caveat: robotics/pick-and-place 중심입니다. CFD/CAE 적용에는 MORSL 같은 skill library를 solver/CAD/mesh/validation primitive로 바꿔야 합니다.
- Action: Read/Cite

### LLM-as-a-Verifier — verifier as a separate scaling axis

- Link: https://arxiv.org/abs/2607.05391
- Code: https://github.com/llm-as-a-verifier/llm-as-a-verifier
- Type: Agent evaluation / verifier framework
- Why it matters: Engineering agents는 “답이 그럴듯한가”보다 artifact가 검증 가능한지가 중요합니다. continuous score, repeated evaluation, criteria decomposition을 verifier scaling axis로 보는 점은 solver-run/report/design-artifact verification에도 참고할 수 있습니다.
- Caveat: LLM verifier는 물리 truth oracle이 아닙니다. CAE/CFD에서는 residual, conservation, mesh quality, regression checks와 결합해야 합니다.
- Action: Watch/Test

## Engineering-AI Pattern

**Trust moves into executable workflow graphs:** 앞으로 좋은 Engineering AI는 모델 호출이 아니라 CAD/mesh/solver/postprocess/validation primitive를 graph로 묶고, case family에서 리허설하며, 실패 위치와 fallback을 남기는 쪽으로 갈 가능성이 큽니다.
