# Daily Research Radar — 2026-05-28

## Highlights

- **오늘의 sciML 신호는 “adaptive/local structure가 보이는 surrogate”입니다.** Courant는 Perceiver 계열 surrogate의 latent가 물리 공간에서 local support와 coherent structure를 갖도록 설계해, CFD surrogate를 단순 black-box regressor보다 hp-refinement/ROM에 가까운 언어로 평가하게 만듭니다.
- **PINN은 평면 PDE를 넘어 geometry-processing surface PDE로 확장 중입니다.** PINNsur는 surface normal neural field와 투영 미분 연산자로 curved surface PDE를 풀어, CAD/B-Rep/mesh 품질이 복잡한 엔지니어링 geometry에서 참고할 만합니다.
- **solver/수치해석 쪽 기본기도 계속 중요합니다.** collocation-DG equivalence 논문은 ML 논문은 아니지만, entropy-stable DG/collocation 계열 surrogate·differentiable solver를 읽을 때 discretization equivalence와 nullspace 처리를 확인하라는 좋은 reminder입니다.
- **agentic engineering workflow는 “도구 연결”보다 “권한·검증·artifact contract”가 핵심으로 이동 중입니다.** ChainCaps, Device Context Protocol, sim-benchmark/OpenFOAM plugin류는 LLM agent가 실제 파일·시뮬레이터·장치에 닿을 때 필요한 안전 경계와 재현성 패턴을 보여줍니다.
- **CAD/CAE open-source 툴은 작지만 빠르게 늘고 있습니다.** FreeCAD MCP, AutoFOAM, ParaView CLI 같은 프로젝트는 아직 early-stage라도 CAD→mesh→solver→visualization 루프를 agent-readable하게 만드는 방향성이 VA 관심사와 잘 맞습니다.

## Added Resources

### SciML, PDE surrogates & numerical methods

- [Courant: a State-Adaptive Perceiver-Based Neural Surrogate with Local Support and Interpretable Field Decomposition](https://arxiv.org/abs/2605.25115) — Perceiver encoder-processor-decoder surrogate에 state-adapted latent queries, random Fourier coordinate embedding, lightweight decoder를 결합합니다. latent가 multiscale geometric specialization과 시간 의존 coherent structure를 보인다는 점이 중요합니다. CFD surrogate 평가에서 field error뿐 아니라 “basis/local support가 물리적으로 해석 가능한가”를 보게 만듭니다. `#SciML` `#surrogate` `#Perceiver` `#local-support`
- [PINNsur: Physics-Informed Neural Networks for PDEs on Curved Surfaces](https://arxiv.org/abs/2605.27308) — surface normal을 neural field로 근사하고, R3 미분을 surface projection으로 표현해 curved/orientable manifold 위 PDE를 푸는 PINN framework입니다. CAD/geometry processing/curved heat diffusion 문제에서 mesh 품질 의존성을 줄이는 방향이라 CAD-to-CAE surrogate와 연결됩니다. `#PINN` `#surface-PDE` `#geometry-processing` `#CAD`
- [A collocation scheme that is equivalent to discontinuous Galerkin discretizations](https://arxiv.org/abs/2605.27327) — summation-by-parts spectral collocation operator가 같은 quadrature 기반 DG semi-discretization과 동등함을 보이고, extraneous nullspace mode 처리까지 논의합니다. ML paper는 아니지만 differentiable PDE solver나 DG 기반 neural surrogate를 설계할 때 discretization contract를 확인하는 데 유용합니다. `#numerics` `#DG` `#collocation` `#PDE-solver`
- [Symbolic Regression via Latent Iterative Refinement](https://arxiv.org/abs/2605.27245) — symbolic tokens와 numerical observations를 공유 latent로 넣고, decoded expression을 반복 re-encoding해 식을 refine하는 LEE framework입니다. SciML에서 governing equation discovery/closure model search를 할 때 one-shot symbolic regression보다 iterative posterior refinement 쪽 신호가 있습니다. `#symbolic-regression` `#equation-discovery` `#SciML`

### Agentic engineering workflow, CAD/CAE tools & safety

- [ChainCaps: Composition-Safe Tool-Using Agents via Monotonic Capability Attenuation](https://arxiv.org/abs/2605.26542) — tool chain에서 값마다 sink-specific capability budget을 붙이고 교집합으로 전파해 “permission laundering”을 막는 MCP proxy 접근입니다. CAD/CAE agent가 파일, solver, 웹, 메시징을 섞을수록 이런 권한 감쇠 모델이 안전한 workflow 설계 언어가 됩니다. `#MCP` `#agent-safety` `#tool-use` `#workflow`
- [Device Context Protocol: A Compact, Safety-First Architecture for LLM-Driven Control of Constrained Devices](https://arxiv.org/abs/2605.26159) — MCP를 microcontroller/physical device 제어로 그대로 내리는 대신, sub-50-byte frame, manifest-level capability/range/unit checks, dry-run evaluation을 둔 protocol 제안입니다. thermal rig, fan/pump/controller 같은 physical engineering loop에 LLM을 붙일 때 “units-as-types + dry-run + bridge reject”가 핵심 패턴입니다. `#device-control` `#MCP-adjacent` `#safety` `#control`
- [FreeCAD MCP Server](https://github.com/JakobThiessen/FreeCad_MCP_Server) — live FreeCAD instance를 MCP server와 XML-RPC bridge로 연결해 Sketcher/Part/PartDesign 자동화와 GUI update를 제공하려는 early-stage 프로젝트입니다. parametric CAD automation 후보로 보되, 실제 CFD geometry에는 dimension/constraint/export validation을 반드시 붙여야 합니다. `#FreeCAD` `#MCP` `#CAD-automation`
- [AutoFOAM](https://github.com/AGN000/AutoFOAM) — natural-language prompt에서 gmsh OCC API로 mesh를 만들고 OpenFOAM v2412 case authoring을 self-refining loop로 자동화하려는 OpenFOAM agent 프로젝트입니다. 과장은 조심해야 하지만, “case setup + mesh generation + anti-collapse defenses”를 한 repo에서 다루는 점은 OpenFOAM agent harness 참고용으로 좋습니다. `#OpenFOAM` `#CFD-agent` `#mesh-generation` `#automation`
- [ParaView-CLI](https://github.com/adricortes/Paraview-CLI) — pvpython server와 lightweight client로 VTK/VTU/OpenFOAM/Exodus/CGNS 등 scientific visualization을 headless/agent-driven하게 다루려는 프로젝트입니다. CFD agent workflow에서 solver 후처리와 figure generation을 GUI 클릭 대신 scriptable artifact로 만드는 방향입니다. `#ParaView` `#visualization` `#CFD-postprocessing` `#agent-tools`

## Best Find of the Day

**Courant**를 오늘의 best find로 봅니다. CFD surrogate에서 “정확도 조금 향상”보다 더 중요한 건 geometry와 state 변화에 따라 latent/basis가 어떻게 조직되는지 볼 수 있는가입니다. Courant의 local support·interpretable field decomposition 주장은 아직 paper-only 신호지만, VA가 관심 있는 complex geometry CFD surrogate, reduced-order modeling, sensor-to-field reconstruction을 연결하는 좋은 평가 축을 줍니다.

## Possible Research Ideas

1. **Local-support surrogate diagnostic:** OpenFOAM duct/heat-sink case에서 FNO/GraphNO/Perceiver-style surrogate를 비교하되, field error 외에 latent activation locality, boundary-layer sensitivity, sensor ablation sensitivity를 같이 본다.
2. **Surface-PINN CAD toy problem:** FreeCAD로 curved shell/pipe surface를 만들고 PINNsur-style projection derivative로 surface heat diffusion을 풀어 FEM baseline과 mesh-quality sensitivity를 비교한다.
3. **Agent-safe CAD→CFD contract:** ChainCaps/DCP/Hylos 관점으로 CAD agent command에 `input authority`, `units`, `range`, `dry_run`, `export_validation`, `rollback` 필드를 붙이고, OpenFOAM case generation 전 gate로 둔다.
4. **Headless post-processing harness:** ParaView-CLI 또는 pvpython wrapper로 OpenFOAM 결과에서 residual plot, field slice, KPI table을 자동 산출하고 sim-benchmark식 verifier가 재실행할 수 있게 만든다.
